The MaybeBoost Hash Library: Functions

The compute_digest Family

These are convenience functions for calculating the message digest of a single range of data. All versions take a HashPolicy model as an explicit template parameter and return the corresponding HashPolicy::digest_type. All the bits in the provided values are used (including the sign bit, if present).

There are three ways to provide the range:

  1. As a pair of iterators
    • compute_digest<HashPolicy>(InIter b, InIter e)
  2. As an iterator and a length
    • compute_digest_n<HashPolicy>(InIter b, size_t n)
  3. As a whole container
    • compute_digest<HashPolicy>(Container const &c)
      when c has begin() and end() member functions
    • compute_digest_n<HashPolicy>(Container const &c)
      when c has begin() and size() member functions
    • compute_digest_data<HashPolicy>(Container const &c)
      when c has data() and size() member functions
    When the Container is a std::basic_string, compute_digest or compute_digest_n delegate to compute_digest_data.

Portability Note: Because the number of bits to use is taken from the size of the iterator's value_type, it's best to use the uint_t<N>::exact types from Boost.Integer rather than directly use fundamental types. Most users are unlikely to need to worry about this, however, as they will be using ranges of bytes on POSIX-compatible machines (where CHAR_BIT is always 8).

Alternative: If you need to specify the number of bits to use from each value, then use the StreamHash directly. This can still be done as an expression as follows: HashPolicy::stream_hash<N>::type().update(b, e).end_message().
