This example compares thread::get_id with this_thread::get_id, and how it plays with detach(). In one spot main attempts to move a std::thread from itself to another scope. Because my bind and tuple haven't been "moved" yet, I had to fake it with an ugly std::ref followed by an inappropriate move.

If one follows the output and source together, as if you were a human source-level debugger, the logic should become clear. Also note that detach is used (twice!) and process termination is guaranteed to be orderly (in this example).
