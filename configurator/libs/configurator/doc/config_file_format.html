Configuration file format

Option
Section
Comments

Option

In fact, configuration file contains a set of data. Minimal unit of data is option. Every option consists of name and value. For example: 
Host = 127.0.0.1
There is one option, with name Host and value 127.0.0.1.

Name and value separated by a some separator, default is =. You can set custom separator (for details, see Configuration file customizing).

Section

Set of options is often corresponds to some conceptual entity, so it useful to combine in the section. For example: 
<Server>
	Host = 127.0.0.1
	Port = 80
</Server>
Section Server corresponds to some server, and options Host and Port regarded not as an independent options, but as parts of conceptual entity "Server".

Of course, you can use arbitrary nesting sections: 
<Server>
	Host = 127.0.0.1
	Port = 80
	
	<Security>
		Admin login	= admin
		Password 	= verystrongpassword
	</Security>
	
	<Reserve>
		<System>
			Log 	= /var/log/system
		</System>
	</Reserve>
	
	<Plugins>
		extended_plugin = sync check browser
	</Plugins>
</Server>
As you see, section's name enclosed in XML-like angle brackets (<---> for section opening, </---> for section closing). This is default format, but you can set custom format (for details, see Configuration file customizing).

The careful reader may notice that option's name (for example, Admin login) and option's value (for example, sync check browser) can contains spaces, except when space/tab using as separator (for details, see Configuration file customizing).

Comments
