Motivation

Many programs (especially servers, daemons, etc.) requires configuration file for work, so it handling is very common task. Actually, this handling add up to four tasks:
  1. obtaining options from configuration file,
  2. check correctness (in all senses of this word) of obtained options,
  3. storing options,
  4. issuance of the options values on demand.
Boost.Configurator provides general solution for this tasks.

Hello, world!

Simplest configuration file: 
Host = 127.0.0.1
Code for work with it: 
#include <boost/configurator/configurator.hpp>

#include <iostream>

BOOST_CONFIGURATOR_OPTION( Host )

int main( int argc, char* argv[] ) {	
    try {
        boost::cf::configurator conf; 
        conf.add< Host >();
        
        conf.load( "/some/path/to/hello_world.conf" );

        std::string host = conf.get_value_of< Host >();
        std::cout << "Value of host: " << host << std::endl;
    } catch ( const std::exception& exc ) {
        std::cerr << exc.what() << std::endl;
    } 

    return 0;
}
Console output: 
Value of host: 127.0.0.1

Features

  • Header-only.
  • Portable (there is only ISO C++ code and Boost C++ libraries).
  • Type-based work with options, without string-identifiers.
  • Option's necessity.
  • Option's default value.
  • Arbitrary nesting of sections.
  • Checks of value's semantics, like correct path, valid IP, etc.
  • Common checks of options and sections, like duplication, incorrection, etc.
  • Single-line and multi-line comments (in C++ style).
  • Options with multi-values.
  • Singleton-variant of core class.
  • Case (in)sensitivity for names of options and sections.
  • Customizing configuration file format.
