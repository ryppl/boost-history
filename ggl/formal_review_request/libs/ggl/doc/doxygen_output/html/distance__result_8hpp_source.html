00148 /*
00149 
00150     From Phil Endecott, on the list:
00151 
00152     You can go further.  If I'm searching through a long list of points to
00153     find the closest to P then I'll avoid the squaring (and conversion to
00154     double if my co-ordinates are integers) whenever possible.  You can
00155     achieve this with a more complex distance proxy:
00156 
00157     class distance_proxy {
00158        double dx;
00159        double dy;
00160        distance_proxy(double dx_, double dy_): dx(dx_), dy(dy_) {}
00161        friend pythag_distance(point,point);
00162     public:
00163        operator double() { return sqrt(dx*dx+dy*dy); }
00164        bool operator>(double d) {
00165          return dx>d
00166              || dy>d
00167              || (dx*dx+dy*dy > d*d);
00168        }
00169     };
00170 
00171     So this is convertible to double, but can be compared to a distance
00172     without any need for sqrt() and only multiplication in some cases.
00173     Further refinement is possible.
00174 
00175 
00176     Barend:
00177     feasable, needs to be templatized by the number of dimensions. For distance it
00178     results in a nice "delayed calculation".
00179     For searching you might take another approach, first calculate dx, if OK then dy,
00180     if OK then the sqrs. So as above but than distance does not need to be calculated.
00181     So it is in fact another strategy.
00182 
00183 
00184 */
