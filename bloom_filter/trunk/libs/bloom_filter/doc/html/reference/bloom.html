Basic Bloom Filter

template <typename T, size_t Size, class HashFunctions = mpl::vector<boost_hash<T, 3> > > class bloom_filter { // exported typedefs typedef T value_type; typedef T key_type; // constructors bloom_filter(); bloom_filter(const std::initializer_list<T>&); // requires C++11 <typename InputIterator> bloom_filter(const InputIterator start, const InputIterator end); // data structure metadata query functions static constexpr size_t bit_capacity() const; static constexpr size_t num_hash_functions() const; double false_positive_rate() const; size_t count() const; bool empty() const; // data structures core ops void insert(const T&); <typename InputIterator> void insert(const InputIterator start, const InputIterator end); bool probably_contains(const T&) const; // auxilliary ops void clear(); void swap(bloom_filter&); // union assign/intersect assign bloom_filter& operator|=(const bloom_filter&); bloom_filter& operator&=(const bloom_filter&); }; // union template <typename T, size_t Size, class HashFunctions> bloom_filter<T, Size, HashFunctions> operator|(const bloom_filter<T, Size, HashFunctions>& lhs, const bloom_filter<T, Size, HashFunctions>& rhs); // intersect template <typename T, size_t Size, class HashFunctions> bloom_filter<T, Size, HashFunctions> operator&(const bloom_filter<T, Size, HashFunctions>& lhs, const bloom_filter<T, Size, HashFunctions>& rhs); template <typename T, size_t Size, class HashFunctions> void swap(bloom_filter<T, Size, HashFunctions>& lhs, bloom_filter<T, Size, HashFunctions>& rhs);

The basic Bloom filter data structure. Supports insertion and query. Allows setting of bit capacity, type, and hash functions at compile-time. This structure is best used when:

  • element removal is not needed
  • the number of insertions can be reliably upper-bounded ahead of time
  • storage requirements are strict

Template Parameters Reference

T
The type used for all Bloom filter operations.
Size
The size in bits of the underlying std::bitset used by the Bloom filter.
HashFunctions
The set of hash functions used by the Bloom filter. Currently required to be an mpl::vector of Hashers.

Returns the current false positive rate based upon the number of hash functions used (k), the number of bits available (m), and the number of bits set (n).

void insert(const InputIterator start, const InputIterator end);
Description
Inserts all elements in range (start, end).
Post-condition
At least n bits and at most (n * num_hash_functions()) bits of the Bloom filter will have been set, where n = distance(start, end).
bool probably_contains(const T&) const;
Description
Queries an element in the Bloom filter, checking each hash function value against the set bits.
Returns
True if all hash functions agree with the bitset; false otherwise.
Warning
It may return a false positive if there are hash collisions - this means that depending on the Size of the Bloom filter and the hash functions used, there is a probability that the Bloom filter will return true even though the element was never inserted.
void clear();

bloom_filter& operator|(const bloom_filter& lhs, const bloom_filter& rhs);
Description
Performs a union.
Returns
Returns a new, stack-allocated Bloom filter that is the result of performing a union between the lhs and the rhs.
Post-condition
The number of bits set in the new filter will be greater than or equal to the number of bits set in max(lhs.count(), rhs.count()).
