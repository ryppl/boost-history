Interlocked Fence

Description

An Interlocked Fence is a template whose instantiations are functions which, when called, imply memory and/or compiler barriers to guarantee order and visibility of unordered or partially ordered operations relative to the call-site. Semantics may be specified by the programmer at compile-time via explicit template arguments.

Concept of
  • Interlocked Memory Fence
  • Interlocked Compiler Fence
