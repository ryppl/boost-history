Consumer

A Consumer is an object which takes in data of a single type one instance at a time and processes it in an implementation defined-manner either sychnronously or asynchronously with respect to the thread it is interfaced with. In Boost.Act, models of this concept may be found as members of models of the Consumer Policy concept.

In order to be considered a valid Consumer, given the variable definitions below, the following expressions must be well-formed and have the indicated effects.

Consumer Policy

A Consumer Policy is a type which is used as a handle to related Consumer types. These Consumer Policys generally act as policies to higher-level templates which choose the appropriate Consumer type based on the knowledge of how the Consumer will be used. In Boost.Act, such types are used as policies during active-qualification.

In order to be considered a valid Consumer Policy, given the variable definitions below, the following expressions must be well-formed and have the indicated effects.
