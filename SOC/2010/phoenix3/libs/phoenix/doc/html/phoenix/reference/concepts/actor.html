Actor

Description

The Actor is the main concept behind the library.

Actors are TR1 function objects and can accept 0 to PHOENIX_LIMIT arguments. An Actor is responsible for setting up the Environment which is then passed to the Evaluator. Every Actor needs to somehow embed the expression it is supposed to evaluate. An Actor itself is required to be a valid Expression.

Note

You can set PHOENIX_LIMIT, the predefined maximum arity an actor can take. By default, PHOENIX_LIMIT is set to 10.
