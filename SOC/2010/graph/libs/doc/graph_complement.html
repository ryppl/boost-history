The result graph is the complement of the input graph g_in, i.e., the graph with the same vertex set of g_in but whose edges are not present in g_in. The reflexive version creates self-loops.

Where Defined

boost/graph/complement.hpp

Parameters

IN: const VertexListGraph& g_in
A directed or undirected graph. The graph type must be a model of Vertex List Graph.
OUT: MutableGraph& g_out
The complement of g_in. The graph type must be a model of Mutable Graph.

Named Parameters

IN: vertex_copy(VertexCopier vc)
This is a function that copies the properties of a vertex in the original graph into the corresponding vertex in the output.
Default: default_vertex_copier which uses the property tag vertex_all to access a property map from the graph.
IN: edge_visitor(EdgeVisitor ev)
This is a function that is called for brand new edges in the output, making it possible to set the edge property.
Default: default_edge_visitor which does nothing.
IN: vertex_index_map(VertexIndexMap i_map)
The vertex index map type must be a model of Readable Property Map and must map the vertex descriptors of g_in to the integers in the half-open range [0,num_vertices(g_in)).
Default: get(vertex_index, g_in). Note: if you use this default, make sure your graph has an internal vertex_index property. For example, adjacenty_list with VertexList=listS does not have an internal vertex_index property.
UTIL/OUT: in_to_out(In2OutMap c)
This maps vertices in the original graph to vertices in the output. Default: an iterator_property_map created from a std::vector of the output graph's vertex descriptor type of size num_vertices(g_in) and using the i_map for the index map.

Complexity

The time complexity is O(V²).
