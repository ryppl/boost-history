Description and Implementation

Modified Bessel functions of order 0 are building blocks of modified Bessel functions of integer order. Moreover, they are commonly used and deserve to be implemented separately at their own right. Most numerical libraries include them as independent functions.

The function domains are  I0(x): (-∞, ∞),  K0(x): (0, ∞).

Note

 I0(x) is even function.

The algorithms consist of minimax rational approximations on finite and infinite intervals. The coefficients of them are from [4] and [16] respectively.

Testing

The test driver calculates relative errors and the following table lists some typical values.

Function float double long double
bessel_i0(x) ~ 10-8  ~ 10-16  ~ 10-20 , 10-19 
bessel_k0(x) ~ 10-8  ~ 10-16  ~ 10-19 , 10-17 

The results are quite robust in the sense that the relative error stays at close to minimum level and does not fluctuate much. The only exception is bessel_k0(x) with long double, whose relative error is 2 or 3 magnitudes higher than the minimum.

Description and Implementation

Bessel functions of order 1 are building blocks of Bessel functions of integer order. Moreover, they are commonly used and deserve to be implemented separately at their own right. Most numerical libraries include them as independent functions.

The function domains are  J1(x): (-∞, ∞),  Y1(x): (0, ∞).

Note

 J1(x) is odd function.

The algorithms consist of minimax rational approximations on root-bracketing intervals for small |x| and Hankel asymptotic expansion for large |x|. The coefficients of them are from [9] and [11] respectively.

Testing

The test driver calculates relative errors and the following table lists some typical values.

Function float double long double
bessel_j1(x) ~ 10-8 , 10-7  ~ 10-16 , 10-15  ~ 10-20 , 10-19 
bessel_y1(x) ~ 10-8 , 10-7  ~ 10-16 , 10-15  ~ 10-18 , 10-17 

When |x| becomes large ( > 104 ), the relative error grows rapidly. |x| = 105 can have relative error 4 or 5 magnitudes higher than that of |x| = 10. Fortunately, this is where the function value approaches zero, so the absolute error is actually quite small.

Description and Implementation

Bessel functions of order n are calculated from Bessel functions of order 0 and Bessel functions of order 1, with the help of recurrence relations.

The function domains are  Jn(x): (-∞, ∞),  Yn(x): (0, ∞).

Note

 Jn(x) is even function when n is even and odd function when n is odd.

The forward recursion itself is stable for n < |x| and unstable for n >= |x|.  Yn(x) is the dominant solution of the recurrence relation, thus the forward recursion is always OK for it. However,  Jn(x) is the minimal solution, so in unstable regions, Miller's algorithm [1] [2] and backward recursion must be used.

Testing

The test driver calculates relative errors and the following table lists some typical values.

Function float double long double
bessel_jn(x) ~ 10-8 , 10-7  ~ 10-16 , 10-15  ~ 10-20 , 10-19 
bessel_yn(x) ~ 10-7  ~ 10-16 , 10-15  ~ 10-19 , 10-17 

When |x| becomes large ( > 104 ), the relative error grows rapidly. |x| = 105 can have relative error 4 magnitudes higher than that of |x| = 10. Fortunately, this is where the function value approaches zero, so the absolute error is actually quite small. n does not seem to have a large impact on accuracy as |x| does.
