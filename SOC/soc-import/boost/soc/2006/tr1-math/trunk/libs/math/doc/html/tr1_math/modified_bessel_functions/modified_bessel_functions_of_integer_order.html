Description and Implementation

Modified Bessel functions of order 0 are building blocks of modified Bessel functions of integer order. Moreover, they are commonly used and deserve to be implemented separately at their own right. Most numerical libraries include them as independent functions.

The function domains are  I0(x): (-∞, ∞),  K0(x): (0, ∞).

Note

 I0(x) is even function.

The algorithms consist of minimax rational approximations on finite and infinite intervals. The coefficients of them are from [4] and [16] respectively.

Testing

The test driver calculates relative errors and the following table lists some typical values.

Function float double long double
bessel_i0(x) ~ 10-8  ~ 10-16  ~ 10-20 , 10-19 
bessel_k0(x) ~ 10-8  ~ 10-16  ~ 10-19 , 10-17 

The results are quite robust in the sense that the relative error stays at close to minimum level and does not fluctuate much. The only exception is bessel_k0(x) with long double, whose relative error is 2 or 3 magnitudes higher than the minimum.

Description and Implementation

Modified Bessel functions of order n are calculated from modified Bessel functions of order 0 and modified Bessel functions of order 1, with the help of recurrence relations.

The function domains are  In(x): (-∞, ∞),  Kn(x): (0, ∞).

Note

 In(x) is even function when n is even and odd function when n is odd.

The forward recursion is always unstable. This is not a problem for  Kn(x) for it is the dominant solution of the recurrence relation. However,  In(x) is the minimal solution, so Miller's algorithm [1] [2] and backward recursion must be used for all x.

Testing

The test driver calculates relative errors and the following table lists some typical values.

Function float double long double
bessel_in(x) ~ 10-8 , 10-7  ~ 10-16  ~ 10-19 , 10-17 
bessel_kn(x) ~ 10-8  ~ 10-16  ~ 10-20 , 10-17 

The results are quite robust in the sense that the relative error stays at close to minimum level and does not fluctuate much. The exception is type long double, whose relative error is 2 or 3 magnitudes higher than the minimum. It is counterintuitive that the most precise type performs worse than float and double. It is not clear whether it's the problem of the expected function values or some other reasons.
