Design Notes

Rationale

The Boost.Explore library is intended primarily as a debugging tool although in some cases it may prove sufficiently capable to provide output good enough for other applications.

  • Ease of use. The interfaces to the library should be as simple as possible reflecting good defaults while allowing more sophisticated options with a small amount of customization.
  • Support user defined types. Since collections can support user defined types Boost.Explore should be capable of printing containers with thoes types.
  • Portability. The library should work with a variety of compilers and platforms.
