definitions

type-only-template-id
A template-id whose template is a type-only-template.
type-only-template
A template whose formal arguments are all types. IOW, a Metafunction except that the nested type typename member is not required.

application_parts

Synopsis

template< 
      typename App
    >
struct application_parts
{
    typedef vector< unspecified > type;
};

Description

A unary Metafunction transforming an n-ary type-only-template-id App into a vector composed of a n-ary Placeholder Expression followed by the n actual arguments to the template in the template-id.

Currently, only values of n in {1..3} are implemented.

Header

#include <boost/mpl/application_parts.hpp>

Model of

Metafunction

Parameters

Parameter Requirement Description
App type-only-template-id A type-only-template application

Expression semantics

For a type-only-template F, and arbitrary types a1,... an:

typedef application_parts<F<a1,a2,...an> >::type g;
Return type:vector.
Semantics:

g is a vector such that

applyn< front<g>::type, pop_front<g>::type >::type

is identical to

F<a1,...an>

Example

struct left{};
struct right{};

BOOST_MPL_ASSERT(( is_same< 
      application_parts<inherit<left,right> >::type
    , vector<inherit<arg<1>,arg<2> >,left,right>::type
    > ));

Metafunction

child_i_depth_j

Synopsis

template
    < class App
    , class Child
    , class Depth
    , class Parts=application_parts<arg<1> >
    >
struct child_i_depth_j
{
    typedef unspecified type;
};

Description

A Metafunction returning the Child::value-th child at depth, Depth::value of an n-ary type-only-template-id, where n > Child:value. Access to children at depth=0 should be provided by the optional Parts parameter.

Currently, only values of n in {1..3} are implemented.

Header

#include <boost/mpl/child_i_depth_j.hpp>

Model of

Metafunction

Parameters

Parameter Requirement Description
App type-only-template-id A type-only-template application
Child Integral Constant Index of child to access, where indices start at 1 (0 corresponds to function)
Depth Integral Constant Depth to go (Depth == 0 corresponds to App )
Parts Placeholder Expression unary placeholder expresssion which, when applied to App, returns its parts, P, such that at_c<P,0> is the function, and at_c<P,i>, for i=1..n are the arguments to that function.

Expression semantics

Given integer literals:

i, such that 1 <= i ;
j, such that 0 <= j ;
ml, such that i <= ml, for l=1..j ;

and given types:

a_k_l, for k=1..,j-1,j+1,ml, l=1..j-1 ;
a_k_j, for k=1..mj ;

and given the following typedef's of type-only-template-id 's:

typedef F_j-1<a_1_j, a_2_j, ..., a_i-1_j,a_i_j,a_i+1_j, ..., a_mj_j> app_j-1;
...
typedef F_1  <a_1_2, a_2_2, ..., a_i-1_2,app_2,a_i+1_2, ..., a_m2_2> app_1  ;
typedef F_0  <a_1_1, a_2_1, ..., a_i-1_1,app_1,a_i+1_1, ..., a_m1_1> app_0  ;

and given the following typedef of a call to child_i_depth_j:

typedef child_i_depth_j<app_0,int_<i>,int_<j> >::type at_i_j;
Semantics:
is_same<app_0,at_i_j>, for j==0  ;
is_same<a_i_j,at_i_j>, otherwise ;

Example

typedef int_<99>     app_3;
typedef prior<app_3> app_2;
typedef prior<app_2> app_1;
typedef prior<app_1> app_0;

BOOST_MPL_ASSERT(( is_same< 
      child_i_depth_j<app_0,int_<1>,int_<3> >::type
    , app_3
    > ));
