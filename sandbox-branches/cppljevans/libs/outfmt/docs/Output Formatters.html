1
Introduction
1.1. The Problem
The C++ Input/Output library provides a mechanism for customizing the way a class is represented on a linear text stream. There are output operations for the basic data types as well as for std::complex and other data types in the standard library.
Having said that, there are no output operations for any of the STL container types. There is a reason for this: how do we represent the list? Different applications would want different ways for the list to be represented, so there is no easy way to specify a default format. What if you need to output a sub-list? Or have the same format across different container types? Or have different formatting for different objects that have the same container type?
1.2. The Solution
Designing a library that can cope with these problems, as well as others not considered here, while remaining efficient is no easy task. There are other attempts to solve these sorts of problems.
The motivation behind using manipulators is that they are already designed into the C++ I/O library and require little overhead in the input and output mechanism. They also give the user maximum choice in how the data being outputted is formatted.
1.3. Definitions
1.3.1. ForwardIterator
A type indicating a forward iterator as defined by C++98, that supports the following operations:
a != b compare two iterators for inequality
++i move to the next item
*i access the item data
1.3.2. Container
A container type that has the following:
read or write? type or method effects
output typename Container::iterator an iterator that is a ForwardIterator type
output typename Container::iterator Container::begin() returns the first element in the container
output typename Container::iterator Container::end() returns "one-past-the-end" for the container type
sequential container input void Container::clear() remove all data in the container
sequential container input void Container::push_back( const T & value ) adds value to the end of the container
1.3.3. Output Facilities
The type is capable of being outputted to a standard ostream. It supports:
std::basic_ostream< CharT, Traits > & operator<<
(
std::basic_ostream< CharT, Traits > & os,
const T & type
);
1.3.4. Input Facilities
The type is capable of being inputted from a standard istream. It supports:
std::basic_istream< CharT, Traits > & operator>>
(
std::basic_istream< CharT, Traits > & is,
T & type
);
1.3.5. Markers
A sequence of characters used between data items. There are three markers currently used by this library: open sequence; close sequence; and separator.
1.3.6. FormatType
This is the type that the markers take. They can be: char, wchar_t, char *, wchar_t * or one of the std::basic_string variants. It is possible to add support for other format types.
1.4. Portability
The code has been succesfully tested on:
  1. MS VC++ 6.0, 7.0 and 7.1 [Note: C-style arrays are only supported within type deduction for MS VC 7.1.]
  2. Borland C++Compiler 5.5.1 and 5.6.4
  3. GNU G++ 3.x (under the i686-pc-cygwin environment)
  4. GNU G++ 2.95.3-5 (under the i686-pc-cygwin environment) [Note: Type deduction does not work, but the rest of the library does.]
  5. Intel 7.1
Most of the code will now work on MSVC 6.0: thanks to Jonathan Turkanis for his help porting the code.

3.1. Sequence Formatters

3.1.1. boost::io::formatlist
boost::io::formatlist
(
ForwardIterator first,
ForwardIterator last
);
This manipulator will format the list specified by the range [first, last). It requires that the elements contained in the sequence have Output Facilities.
[Example:
int i[ 4 ] = { 1, 2, 3, 4 };
std::cout << boost::io::formatlist( i, i + 4 ); // output: [ 1, 2, 3, 4 ]
]
The manipulator conforms to the format object generator guidelines in section [6.1.].
3.1.2. boost::io::format
boost::io::format
(
Container & c
);
This manipulator takes a sequential container as its argument. It outputs the range [begin(), end()), and requires that the elements in the sequence have Output Facilities.
[Example:
std::vector< int > v;
// ...
std::cout << boost::io::format( v ); // output: [ 3, 6, 9, 12 ]
]
The manipulator conforms to the format object generator guidelines in section [6.1.].
3.1.3. boost::io::formatrange
boost::io::formatrange
(
std::pair< ForwardIterator, ForwardIterator > & ip
);
This manipulator takes an iterator range as its argument in the form of a pair of iterators. It outputs the range [ip.first, ip.second), and requires that the elements in the sequence have Output Facilities.
[Example:
int i[ 4 ] = { 1, 2, 3, 4 };
// construct an iterator range in a std::pair object
std::pair< int *, int * > ip = std::make_pair( i, i + 2 );
std::cout << boost::io::formatrange( ip ); // output: [ 1, 2 ]
]
The manipulator conforms to the format object generator guidelines in section [6.1.].

Formatting Options
examples: libs/outfmt/examples/format-options.cpp
libs/outfmt/examples/formatter.cpp
The formatting facilities are included indirectly when you include other files in the library. They are used in the implementation of the output facilities of the library, including boost::io::formatlist.
These facilities control what is output at the start of a sequence, as a separator between elements and at the end of the sequence.
4.1. Format Traits
The format traits contain the default values for pair types ( boost::io::detail::pair_traits ) and sequence types ( boost::io::detail::format_traits ). These have the following form:
template< typename CharT >
struct format_traits
{
static const CharT open_default = implemenation defined;
static const CharT close_default = implemenation defined;
static const CharT separator_default = implemenation defined;
};
boost::io::detail::pair_traits provides the default formatting:
( ..., ... )
boost::io::detail::format_traits defaults the default formatting:
[ ..., ... ]
Both are defined for char, wchar_t, char * and wchar_t *. If partial template specialisation is supported, it is also specialized for std::basic_string, otherwise std::string and std::wstring are supported.
4.2. Open/Close Formatter
4.2.1. boost::io::openclose_formatter_t
template
<
typename FormatType,
class ReferenceType,
class FormatTraits = boost::io::detail::format_traits< FormatType >

boost::io::openclose_formatter_t defines the formatting facilities for wrapping around a output type. It defines the following types:
format_type the character/string type used in the formatting

The ReferenceType parameter is used to specify the return type on the format( ... ) function, allowing further access to the derived object. This allows you to apply formatting onto a manipulator, like the example in section [4.2.1.3.].

5
Format Objects
5.1. Requirements
5.1.1. Output Facilities
A format object that is intending to provide output of a type to a stream must define a write function that accepts that type by defining:
template< typename T, class OutputStream >
inline OutputStream & write( OutputStream & os, const T & value ) const
(
// ...
return( os );
);
T is the type being outputted using this format object.
[Note: The type T here is a standin for the actual type being used. For example, the format object can be implemented to accept std::complex< T >, allowing complex types to be formatted.]
OutputStream is the stream type. This is not std::basic_ostream< CharT, Traits > here to simplify the implementation of the operator (it is easier to type!). If you want to use std::basic_ostream< CharT, Traits > instead, you can, but this is not advised for compatibility with older libraries that do not support std::basic_ostream.
The operator is const because the format object is not modified. For state-based formatting that requires modification of the data, like position information, there is a special format object. See libs/outfmt/examples/john-torjo.cpp for an example of this.
5.1.2. Input Facilities
A format object that is intending to provide input of a type to a stream must define a read function that accepts that type by defining:
template< typename T, class InputStream >
inline bool read( InputStream & is, T & value ) const
(
// ...
return( true );
);
Where true indicates a successful read operation and false indicates there was a problem.
5.1.3. Formatting
In order to be chained (when forming complex format objects for nested types), the format object must define the type format_type to be the string or character type to be used for markers.
In order to allow the user to customise the markers, a format object should inherit from a formatter. See section [4.] for more information on formatters. [Note: Formatters define the type format_type.]
5.2. Simple Types
5.2.1. boost::io::basic_object
boost::io::basic_object is used as a generic format object, providing no specific formatting.
5.2.2. boost::io::wrapped_object
boost::io::wrapped_object inherits its formatting from boost::io::openclose_formatter_t, allowing specific formatting to be added around a type. This would be useful for things like formatting a complex type as XML data.
5.3. N-ary Types

5.4. Sequences
5.4.1. boost::io::detail::list_object
boost::io::detail::list_object is a helper class, inheriting its formatting from boost::io::formatter_t. It is used to output sequential lists of the form [first, last).
boost::io::detail::list_object is not technically a format object, since it has the function operator:
template< typename ForwardIterator, class OutputStream >
inline OutputStream & operator()
(
OutputStream & os,
ForwardIterator first,
ForwardIterator last
);
The class is used to help the implementations of boost::io::array_object, boost::io::container_object and boost::io::range_object.
5.4.2. boost::io::array_object
examples: libs/outfmt/examples/output-2D.cpp
libs/outfmt/examples/output-3D.cpp
boost::io::array_object inherits its functionality from boost::io::detail::list_object, supporting the output of C-style arrays of the form const T *.
To facilitate the output of arrays, it defines two special function operators:
operator()( long len );
Specify the range [0, len) to be outputted.
operator()( long off, long len );
Specify the range [off, off + len) to be outputted.
5.4.3. boost::io::container_object
examples: libs/outfmt/examples/output-2D.cpp
boost::io::container_object inherits its functionality from boost::io::detail::list_object, supporting the output of containers as defined in section [1.3.2.].
5.4.4. boost::io::range_object
boost::io::range_object inherits its functionality from boost::io::detail::list_object, supporting the output of an iterator range. The iterator range is a pair of iterators are defined in section [1.3.1.].
This formatter allows ranges to be constructed from a container, a std::pair, a set of values and a C-style array through the use of the boost::io::range function.
[Note: This is different from boost::io::pair_object, that formats the elements in the pair as the first and last elements in a list.]
5.5. Input/Output Manipulators
5.5.1. boost::io::formatob_t
boost::io::formatob_t is the class that is used to implement the boost::io::formatob manipulators. It is a hybrid between a format object and a manipulator, that inherits it's output capabilities from another format object. See section [3.2.] for more details.
This is used to construct the Sequence Formatters: they are really helper functions that represent a more complex format object.
5.6. State Information
examples: libs/outfmt/examples/john-torjo.cpp
5.6.1. boost::io::state_object
boost::io::state_object does not inherit from a formatter class, and as such is not a standard format object. It is a helper that will bind a state object and a format object without the need for casts in the user code (see the example for a demonstration of this).
5.6.1.1. State Objects
A state object is a type of function object that has the form:
template< class OutputStream >
inline OutputStream & operator()
(
OutputStream & os
);
A state object should only output state information to the stream.
6
Format Object Generators
6.1. Requirements
6.1.1. Default
The default format object generator has the form:
name()
where name is the format object name for its generator, unless otherwise noted.
It uses the char * format type, and the default sub-element format object.
6.1.2. FormatType Specifier
This format object generator has the form:
nameex< FormatType >()
where name is the format object name for its generator, unless otherwise noted.
It uses the format type specified by FormatType, and the default sub-element format object.
6.1.3. Format Object Specifier
This format object generator has the form:
name( const FormatObject & )
where name is the format object name for its generator, unless otherwise noted.
It uses the format type specified by FormatObject::format_type, and uses FormatObject to format its sub-elements.
6.2. Simple Types
6.2.1. boost::io::basicfmt
This generator function creates a boost::io::basic_object object. It does not meets the requirements specified in section [6.1.]: it does not have any variants.
6.2.2. boost::io::wrappedfmt
This generator function creates a boost::io::wrapped_o object. It meets the requirements specified in section [6.1.].

6.5. State Information
examples: libs/outfmt/examples/john-torjo.cpp
6.5.1. boost::io::statefmt
This generator function creates a boost::io::state_object object. It meets the requirements specified in section [], but has slightly different semantics.
statefmt< StateObject, bool pre >()
This version takes a state object and specifies whether the state object is outputted before ( pre = true ) or after ( pre = false ) the format object hosted by boost::io::state_object.

This version allows the user to specify the format type used by the format objects.
statefmt< StateObject, bool pre >( const Outputter & o )
This version controls the format object used to format the data type.
