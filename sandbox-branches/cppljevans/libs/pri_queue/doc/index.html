Boost Priority Queues

A priority queue is a queue that enforces some ordering policy on its elements during its entire lifetime. The basic priority queue provides two operations: a push operation, which inserts an element, and a pop operation, which removes the minimum element according to the queue's ordering policy. Even the simple queue can be thought of as a priority queue where the minimum element is the one least recently inserted. Most priority queue implementations, however, impose external ordering policies, which are not intrinsic to either the queues or the elements themselves. Most priority queues also provide one or more update operations for maintaining the integrity of their internal structures when their ordering policies change. This ability to update a priority queue is vital for applications such as thread scheduling and graph algorithms.

Boost provides the following priority queues. All of them model the Buffer concept. With the exception of the fenced_priority_queue, they also model the Updatable Buffer concept.

  • mutable_queue
  • fibonacci_heap

The indirect_cmp function object adaptor provides the actual external ordering on a priority queue's elements. Without indirect_cmp, the priority queue's update operation is next to useless.

The priority queue is a standard data structure, and is discussed in all algorithm books worth their ink. For more ideas on their general usage, see http://www.leekillough.com/heaps/.
