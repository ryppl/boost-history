Macro BOOST_MIRROR_REG_TYPE

BOOST_MIRROR_REG_TYPE — This macro is used to register a type, that is defined inside of a namespace, with Mirror. After successful registration the type can be reflected.

Synopsis

BOOST_MIRROR_REG_TYPE(NAMESPACE, TYPE)

Description

The BOOST_MIRROR_REG_TYPE macro registers a type with the Mirror library.

To register a type defined in the global scope use the BOOST_MIRROR_REG_TYPE_GLOBAL_SCOPE or the BOOST_MIRROR_REG_TYPE_GS macro. To register a typedefined type use one of the BOOST_MIRROR_REG_TYPEDEF_GLOBAL_SCOPE, BOOST_MIRROR_REG_TYPEDEF_GS or BOOST_MIRROR_REG_TYPEDEF macros. To register a type that is defined inside of a class use either the BOOST_MIRROR_REG_TYPE_EMBEDDED or the BOOST_MIRROR_REG_TYPE_EMB macro.

Note
The registering must be done in the ::boost::mirror namespace.

The first parameter is the full name (including the leading double colon) of the namespace where the type is defined. The second parameter is the name of the type (without the namespace name).

To register a type bar defined in a named namespace foo, 

namespace foo {
  struct bar { };
} // namespace foo

do this:

// note that the registering must be done
// in the boost::mirror namespace

namespace boost {
namespace mirror {

// registers ::foo
BOOST_MIRROR_REG_NAMESPACE((foo))
// note that the full namespace name including the leading 
// double colon must be passed as the first argument
BOOST_MIRROR_REG_TYPE(::foo, bar)

} //namespace mirror
} //namespace boost

Afterwards the types can be reflected with the BOOST_MIRRORED_TYPE macro:

BOOST_MPL_ASSERT((
	reflects_type< BOOST_MIRRORED_TYPE(::foo::bar) >
));
BOOST_MPL_ASSERT((
	reflects_namespace< BOOST_MIRRORED_TYPE(::foo::bar)::scope >
));
BOOST_MPL_ASSERT((
	reflects_global_scope< BOOST_MIRRORED_TYPE(::foo::bar)::scope::scope >
));

The Mirror library has several preregistered types:

namespace boost {
namespace mirror {

BOOST_MIRROR_REG_TYPE(::std, string)
BOOST_MIRROR_REG_TYPE(::std, wstring)
BOOST_MIRROR_REG_TYPE(::boost::tuples, tuple)

} //namespace mirror
} //namespace boost
