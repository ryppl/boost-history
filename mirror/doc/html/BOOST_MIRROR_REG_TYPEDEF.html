Macro BOOST_MIRROR_REG_TYPEDEF

BOOST_MIRROR_REG_TYPEDEF — This macro is used to register a typedef, that is defined inside of a namespace, with the Mirror library. After successful registration the typedef can be reflected with Mirror.

Synopsis

BOOST_MIRROR_REG_TYPEDEF(NAMESPACE, TYPEDEF)

Description

The BOOST_MIRROR_REG_TYPEDEF macro registers a typedef defined in a namespace with the Mirror library. To register a typedef from the global scope use one of the BOOST_MIRROR_REG_TYPEDEF_GLOBAL_SCOPE or BOOST_MIRROR_REG_TYPEDEF_GS macros. To register a type defined in the global scope use the BOOST_MIRROR_REG_TYPE_GLOBAL_SCOPE or the BOOST_MIRROR_REG_TYPE_GS macro. To register a type defined in a namespace use the BOOST_MIRROR_REG_TYPE macro. To register a type that is defined inside of a class use either the BOOST_MIRROR_REG_TYPE_EMBEDDED or the BOOST_MIRROR_REG_TYPE_EMB macro.

Note
The registering must be done in the ::boost::mirror namespace.

The first parameter is the full name (including the leading double colon) of the namespace where the type is defined. The second parameter is the typedefined name of the type (without the namespace name).

To register a typedefined type bar defined in a named namespace foo, 

namespace foo {
  typedef double bar;
  
} // namespace foo

do the following:

// note that the registering must be done
// in the boost::mirror namespace

namespace boost {
namespace mirror {

// registers ::foo
BOOST_MIRROR_REG_NAMESPACE((foo))
// note that the full namespace name including the leading 
// double colon must be passed as the first argument
BOOST_MIRROR_REG_TYPEDEF(::foo, bar)

} //namespace mirror
} //namespace boost

Afterwards the type can be reflected with the BOOST_MIRRORED_TYPEDEF macro:

BOOST_MPL_ASSERT((
	reflects_type< BOOST_MIRRORED_TYPEDEF(::foo, bar) >
));
BOOST_MPL_ASSERT((
	reflects_namespace< BOOST_MIRRORED_TYPEDEF(::foo, bar)::scope >
));
BOOST_MPL_ASSERT((
	reflects_global_scope< BOOST_MIRRORED_TYPEDEF(::foo, bar)::scope::scope >
));

Mirror distinguishes between types and typedefs, thus the meta_type reflecting a typedefined type is different from the meta_type reflecting the aliased type: 

BOOST_MPL_ASSERT_NOT((
	::boost::is_same<
		 BOOST_MIRRORED_TYPE_NS(::foo, bar),
		 BOOST_MIRRORED_TYPEDEF(::foo, bar) 
	>
));

The name getters of the meta_type reflecting the typedef return the typedefined name instead of the name of the aliased type: 

	// the typedef'd name
	assert(
		BOOST_MIRRORED_TYPEDEF(::foo, bar)::base_name() == 
		BOOST_CTS_LIT("bar")
	);
	// the name of the aliased type
	assert(
		BOOST_MIRRORED_TYPE_NS(::foo, bar)::base_name() == 
		BOOST_CTS_LIT("double")
	);
	// full names
	assert(
		BOOST_MIRRORED_TYPEDEF(::foo, bar)::full_name() == 
		BOOST_CTS_LIT("::foo::bar")
	);
	assert(
		BOOST_MIRRORED_TYPE(::foo::bar)::full_name() == 
		BOOST_CTS_LIT("double")
	);

The Mirror library has several preregistered typedefs, for example:

namespace boost {
namespace mirror {

BOOST_MIRROR_REG_TYPEDEF(::boost::cts, bchar)
BOOST_MIRROR_REG_TYPEDEF(::boost::cts, bstring)

} //namespace mirror
} //namespace boost
