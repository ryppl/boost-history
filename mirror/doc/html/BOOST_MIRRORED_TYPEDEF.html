Macro BOOST_MIRRORED_TYPEDEF

BOOST_MIRRORED_TYPEDEF — This macro is used to reflect a typedef-ined type and expands into a specialization of the meta_type template, which provides meta-data about the reflected type. To reflect a base type (not a typedef-ined) use the BOOST_MIRRORED_TYPE macro. To reflect the type of an expression use the BOOST_MIRRORED_TYPEOF macro.

Synopsis

BOOST_MIRRORED_TYPEDEF(FULL_NAMESPACE_NAME, TYPEDEF)

Description

The BOOST_MIRRORED_TYPEDEF macro reflects a type that is registered with the Mirror library as a typedef-ined type. The difference between this macro and the BOOST_MIRRORED_TYPE macro is, that BOOST_MIRRORED_TYPEDEF returns a meta_type specialization which is aware of the typedef-ined name and the scope in which the type is typedef'd. For example the size_t is a typedef from the ::std namespace and is usually defined as 

namespace std {
typedef unsigned int size_t;
} // namespace std

or 

namespace std {
typedef unsigned long size_t;
} // namespace std

the full_name function called on meta_type expanded from BOOST_MIRRORED_TYPEDEF 

	BOOST_MIRRORED_TYPEDEF(::std, size_t)::full_name()

returns "::std::size_t", while calling 

	BOOST_MIRRORED_TYPE_NS(::std, size_t)::full_name()

or 

	BOOST_MIRRORED_TYPE(::std::size_t)::full_name()

returns "unsigned int" or "unsigned long".

Examples
Basic usage
BOOST_MIRRORED_TYPEDEF(::boost::cts, bstring)::base_name();

BOOST_MPL_ASSERT((
	reflects_global_scope<
		BOOST_MIRRORED_TYPEDEF(::std, size_t)::scope::scope
	>
));

BOOST_MPL_ASSERT_NOT((
	reflects_global_scope<
		BOOST_MIRRORED_TYPEDEF(::boost::cts, bstring)::scope
	>
));

BOOST_MIRRORED_TYPEDEF(::boost::cts, bstring)::scope::full_name();

assert(
	BOOST_MIRRORED_TYPEDEF(::std, size_t)::full_name() !=
	BOOST_MIRRORED_TYPE(::std::size_t)::full_name()
);

assert(
	BOOST_MIRRORED_TYPEDEF(::boost::cts, bstring)::full_name() !=
	BOOST_MIRRORED_TYPE_NS(::boost::cts, bstring)::full_name()
);
