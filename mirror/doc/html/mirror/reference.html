Registering

Constructs like namespaces, types, typedefs, classes, class member attributes, base classes, etc. need to be registered before they can be reflected or otherwise used. To simplify the registration process and to make it more convenient, the Mirror library provides several preprocessor macros described in greater detail in the references pages accessible through the links provided below.

Preprocessor macros are frowned upon and not without reasons, but they are helpful in many situations non the less and with little caution don't cause any problems. One of the reasons for using macros for registering various things in Mirror is, that they hide rather complex definitions of specializations of various templates and other details and are much more readable and understandable. Another reason for using macros is, that mirror needs to "remember" the name of the namespace, type, class, etc. and this is where the preprocessor's stringization operator is extremly useful as it removes the necessity to explicitly pass the name in a c-string literal.

The registering macros are listed in the following table together with brief notes and links to reference pages for the individual macros. As a general rule of the thumb these macros should be used in the boost::mirror namespace.

Table 1.1. Registering macros

Macro Purpose Notes
BOOST_MIRROR_REG_NAMESPACE Registers a namespace  
BOOST_MIRROR_REG_TYPE Registers a type defined inside of a named namespace  
BOOST_MIRROR_REG_TYPE_GS Registers a type defined in the global scope This macro is equivalent to the BOOST_MIRROR_REG_TYPE_GLOBAL_SCOPE.
BOOST_MIRROR_REG_TYPE_GLOBAL_SCOPE Registers a type defined in the global scope This macro is equivalent to the BOOST_MIRROR_REG_TYPE_GS.
BOOST_MIRROR_REG_TYPEDEF Registers a typedef defined inside of a named namespace  
BOOST_MIRROR_REG_TYPEDEF_GS Registers a typedef from the global scope This macro is equivalent to the BOOST_MIRROR_REG_TYPEDEF_GLOBAL_SCOPE.
BOOST_MIRROR_REG_TYPEDEF_GLOBAL_SCOPE Registers a typedef from the global scope This macro is equivalent to the BOOST_MIRROR_REG_TYPEDEF_GS.
BOOST_MIRROR_REG_TYPE_EMB Registers a type nested inside of a class This macro is equivalent to the BOOST_MIRROR_REG_TYPE_EMBEDDED.
BOOST_MIRROR_REG_TYPE_EMBEDDED Registers a type nested inside of a class This macro is equivalent to the BOOST_MIRROR_REG_TYPE_EMB.

Reflecting

The reflecting macros are listed in the following table together with brief notes and links to reference pages for the individual macros.

Table 1.2. Reflection macros

Macro Purpose Notes
BOOST_MIRRORED_NAMESPACE Reflects a namespace Expands into a meta_namespace specialization providing meta-data about the reflected namespace.
BOOST_MIRRORED_GLOBAL_SCOPE Reflects the global scope Expands into a meta_namespace specialization providing meta-data about the global scope.
BOOST_MIRRORED_TYPE Reflects a type Expands into a meta_type specialization providing meta-data about a type.
BOOST_MIRRORED_TYPE_NS Reflects a type Expands into a meta_type specialization providing meta-data about a (nested) type. This version of the BOOST_MIRRORED_TYPE macro takes the scope of the type and the base type name as two separate arguments.
BOOST_MIRRORED_TYPEDEF Reflects a typedefined type Expands into a meta_type specialization providing meta-data about a (nested) typedef.
BOOST_MIRRORED_TYPEDEF_GS Reflects a global-scope typedef Expands into a meta_type specialization providing meta-data about a typedef defined in the global scope.
BOOST_MIRRORED_TYPEDEF_GLOBAL_SCOPE Reflects a global-scope typedef Expands into a meta_type specialization providing meta-data about a typedef defined in the global scope. This macro is equivalent to BOOST_MIRRORED_TYPEDEF_GS.
BOOST_MIRRORED_TYPEOF Reflects the type of an expression Expands into a meta_type specialization providing meta-data about the type of the expression given as an argument.
BOOST_MIRRORED_CLASS Reflects a class Expands into a meta_class specialization providing meta-data about a class.

Meta Objects

The reflection macros listed above, expand into template class instantiations that provide meta-data about the reflected namespaces, types, classes, etc. These template instantiations are commonly referred to as MetaObjects. Some other MetaObjects can be accessed by applying the intrinsic meta-functions on other MetaObjects.

Table 1.3. Meta objects

Meta Object Purpose Notes
meta_namespace Reflects a namespace Provides information about the scope of the namespace and its name.
meta_type Reflects a type Provides information about the scope of the type and its name.
meta_class Reflects a class Provides information about the inheritance of the class and about its member attributes.
meta_inheritance Reflects the inheritance of a single base class of another class Provides information about the access specifier, the type of inheritance (virtual vs. regular) and about the base class.
meta_class_attribute Reflects a single member attribute of a class Provides information about the type and position of the attribute, its name and allows to get and set values of the attribute.
meta_class_attributes Reflects own member attributes of a class Instantiations of meta_class_attributes are models of the MetaObjectSequence concept and contain meta_class_attributes reflecting own member attributes of a class, i.e. those defined by the class, not including the inherited ones.
meta_class_all_attributes Reflects all member attributes of a class Instantiations of meta_class_all_attributes are models of the MetaObjectSequence concept and contain meta_class_attributes reflecting all member attributes of a class, including the inherited ones.

Traits

The trait templates can be used to query the properties of the MetaObjects. Each trait template inherits either from the ::boost::true_type or ::boost::false_type to indicate whether the inspected MetaObject has or has not the trait.

Table 1.4. Traits

Trait template Purpose
reflects_namespace Checks whether the given MetaObject reflects a namespace, i.e. whether it is a meta_namespace.
reflects_global_scope Checks whether the given MetaObject reflects the global scope, i.e. whether it is a meta_namespace specialization providing meta-information about the global scope.
reflects_type Checks whether the given MetaObject reflects a type, i.e. whether it is a meta_type.
reflects_class Checks whether the given MetaObject reflects a class, i.e. whether it is a meta_class.
reflects_attribute Checks whether the given MetaObject reflects a class member attribute, i.e. whether it is a meta_class_attribute.
reflects_inheritance Checks whether the given MetaObject reflects a base class inheritance, i.e. whether it is a meta_inheritance.
reflects_virtual_inheritance Checks whether the given MetaObject reflects a virtual class inheritance, i.e. whether it is a meta_inheritance specialization providing meta-information about a base class that is inherited virtually.
