Basic

Pack Pointer was designed to be easy to use and to be versatile in terms of object variants it can refer to. Its only requirement is limited to the usage of a special type needed to instanciate objects referred to. For example:

  
  pack_ptr<int> p = new pack<int>(11);

Will instanciate a special object pack<int> having an integer as one of its member. The pointer to the object is then passed to the pack_ptr<int> that will manage its existence and later destroy and deallocate it when it is found to be no longer referenced.

Variants of the object type can also be used. In order to instanciate an array of integer for example, all that will be needed is to change the type to the following:

  
  pack_ptr<int[10]> p = new pack<int[10]>();
See the following example for different cases of its usage.

Advanced

When STL containers are made out of objects having members of type pack_ptr<T> then it will be desirable to have all of the allocated nodes and its members part of the same pack. This way all of the newly allocated pack_ptr<T> will be referring to the same pack header. In order to do so, we must tell the container in question a new allocator that will be used to instanciate the nodes. This allocator is called: pack_allocator<T>. For example:

  
  list< pack_ptr<int>, pack_allocator< pack_ptr<int> > > c;
  c.push_back(new pack<int>(11));
