Introduction

Pack Pointer introduces the concept of pack where it is defined to be a one or many memory blocks referencing each other allocated on the heap pointed to by one or many pack_ptr<T>s allocated on the stack or the data segment (non heap) of the application. Each pack is consequently composed of a list of all memory blocks constituting it and a counter tracking the number of times the pack is referenced by a pack_ptr<T> from the stack or the data segment. Cyclicism

Therefore whenever a pack counter finds out the number of pack_ptr<T> referencing it to be zero, the deallocation of each memory block composing the pack is enforced. Thus whether the pack was composed of memory blocks referencing each other in a cyclic way or not, all of them will be subject to destruction and deallocation indifferent from the cyclicism problem presented by the reference counters. Initialization

A pack_ptr<T> is initialized differently depending on the memory segment it is being instanciated on. If it is found to be residing the stack or the data segment then a new pack counter will also be instanciated. If it is found to be living on the heap then a more complicated mechanism is involved to track the last memory block that was allocated in the same thread, find its address and stack up the pack it will be part of until the first pack_ptr<T> found on the stack is or data segment is initialized, where the new pack counter will be shared with all of the newly allocated blocks of memory referencing each other.

Union

A pack will be unified with another pack whenever one of its member is found to reference a member from a different pack. The counter from the first pack will be added to the counter from the second pack and a redirection of the first counter to the second will occur so that all elements from the first pack make direct use of the second counter. Once two packs are unified they will stay unified even if the new reference responsible for unifying the packs is once again disjointed.
