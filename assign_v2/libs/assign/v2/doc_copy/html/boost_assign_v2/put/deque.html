Deque

Header 

#include<boost/assign/v2/put/deque.hpp>

Describe

In place construction of a de-que container that has the adapter interface.

Synopsis 

namespace put_aux{

Example

Integers 

{
    typedef as2::result_of::deque<int>::type C;
    C empty_cont = as2::deque<int>( as2::_nil );

    assert( empty_cont( 72 )( 31 )( 48 ).front() == 72 );
    assert( as2::deque<int>( 72 )( 31 )( 48 ).back() == 48 );
}
typedef as2::result_of::csv_deque<int>::type C;
{
    C cont = as2::csv_deque( 72, 31, 48 );

    assert( cont.front() == 72 );
    assert( cont.back() == 48 );
}
{
    C cont = as2::csv_deque( 72, 31 )( 48 );

    assert( cont.front() == 72 );
    assert( cont.back() == 48 );
}

This seeming oddity is possible thanks to result_of::csv_deque<int>::type == result_of::deque<int>::type

String literal 

typedef std::string str_;
assert(
    str_( as2::deque<const char*>( "x" )( "y" )( "z" )[1] ) == "y"
);
assert(
    str_( as2::csv_deque( "x", "y", "z" )[1] ) == "y"
);

Note

The examples above are minimal to avoid redundancies with those of put(), which illustrate specific features of the adapter interface, which is common to both. These examples are easy to translate to deque<>()
