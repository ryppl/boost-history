#include <boost/assign/v2/put/modulo.hpp>

Description

Interface for overriding the components of the implementation.

Semantics

Notation

Symbol

Description

param

A user-specified parameter as rhs of operator%()

Expressions

Expression

Result

put-implementation % param

An implementation with a new type or state

This extends without difficulty to an arbitrary number of modulo parameters. In other words, the mechanism is composable.

There are two pre-defined values for param, the type of _fun and that of _std, which are described in the following sections.

Parameters

Parameter

Description

param

A name identifying a custom implementation

Requirements

In each step below, the user is required to write a piece of code within namespace boost::assign::v2. The notation typename..., here, is not a variadic template parameter. It means that the choice of template parameters are unspecified.

  1. Define 
    namespace put_aux{

   	template<typename Arg>
   	struct param
   	{
   		// Constructor
       	unspecified arg()const; 
   	};

}// put_aux
  2. Define 
    namespace put_aux{

   	struct param_keyword
   	{
   		template<typename Arg>
       	param<Arg> operator=(Arg const&); 
   	};

}// put_aux
    where Arg is the type of param_arg.
  3. Define 
    namespace modifier_tag{

   	template<typename ...> struct param{};

}
  4. Define 
    namespace put_parameter{

   	template<typename Arg>
   	struct param{

           template<typename OldFun, typename OldTag>
           struct apply; // must derive from put_parameter::traits<>

   	};

}
  5. Specialize the frame component 
    namespace put_aux{
	
	template<typename ...>
	struct modifier<modifier_tag::param<...> >;
	
}
    such that its constructor takes as argument the result of ( _param = param_arg ).arg().
Note

The template parameter Arg may be unnecessary. Refer to the examples below, to learn how to do away with it

Semantics 

BOOST_ASSIGN_V2_PUT_MODIFIER_EXT_XXX(param)

Expands to 

namespace result_of_modulo{
    template<typename D, typename Arg>
    struct param
        : ::boost::mpl::apply1<
            result_of_modulo::ext<D>
            , put_parameter::param<Arg>
        >
    {};
}
namespace put_aux{

    template<typename C, typename F, typename Tag, typename D, typename Arg>
    typename result_of_modulo::param<D, Arg>::type
    operator%(crtp<C, F, Tag, D> const& lhs, put_aux::param<Arg> const& rhs);

}// put_aux
namespace{
    put_aux::param_keyword const _param;
}

such that

Expression

put-implementation % (_param = param_arg)

are valid expressions.

Semantics

Expression

Result

(_std = tag() )

Instance of put_aux::modulo_std<tag>

put-implementation % (_std = tag() )

Replaces the internal modifier of put-implementation by an instance of put_aux::modifier<tag>

Concept

Semantics

A class tag models concept Std if class put_aux::modifier<tag> satisfies:

Overloads of impl()

Implementation

Invariant 

void impl(C& cont, T&& t)const;

Passes std::move( t ) to cont's interface.

cont.size() is increased by one. 

void impl(C& cont, T& t)const;

Passes t (by reference) to cont's interface.

cont.size() is increased by one.
