Convention

We add to our conventions that put-implementation denotes an object returned by keyword such as _csv_put, and param denotes a user-specified parameter.

Description

The put framework relies on traits of the container to be modified to determine the appropriate implementation. This has the advantage that the same and compact interface works across different container categories, whether Sequence, Associative or adapters. To allow the user to override the implementation, however, this library provides a composable parameter passing mechanism.

Synopsis

Expression

Result

put-implementation % param

An implementation with a new type or state

_fun

Header 

#include <boost/assign/v2/put/generic/new_fun.hpp>

Synopsis

Expression

Result

put-implementation % (_fun = f)

Replaces the internal functor of the lhs by f

Example 

std::vector<int> cont;
boost::for_each(
    cont | (
        as2::_csv_put % (
            as2::_fun = (2 * bl::_1)
        )
    )( -1, 0, 1 ),
    os << bl::_1 << ' '
); // outputs -2 0 2

Modifier

Standard
Extended
Header 

#include <boost/assign/v2/put/modifier.hpp>

Description

This section documents the mechanisms by which a passed value, t, is put in a container, cont. These mechanisms are of two kinds, standard, and extended.

Standard
Header 

#include <boost/assign/v2/put/modifier/std.hpp>

Description

A standard modifier is charaterized by an implementation that, given a passed value, t, inserts it in the destination container, cont, by calling a member function, cont.mf( t ).

Example 

boost::for_each(
       (
           as2::deque<int>( as2::_nil ) % as2::_push_front
       )( -1 )( 0 )( 1 ),
       os << bl::_1 << ' '
);// outputs 1 0 -1

_repeat
Header 

#include <boost/assign/v2/put/modifier/ext/repeat.hpp>

_repeat = n

where implementation-defined is the implementation prior to passing the parameter.

_lookup
Header 

#include <boost/assign/v2/put/modifier/ext/lookup.hpp>

Synopsis

Let key denote an Associative container's key.

Expression

Implementation

_lookup = f

cont[ key ] = f( cont[ key ] )

Example 

std::map<const char*,int> cont;
cont["x"] = -1; cont["y"] = 0; cont["z"] = 1;
cont | (
    as2::_csv_put % ( as2::_lookup = ( bl::_1 +=2 ) )
)( "z", "x", "y" );
assert( cont["y"] == 2 );

Customization
Header 

#include <boost/assign/v2/put/modifier/ext/xxx.hpp>

Note

This header has to be explicitly included

Description

We want to create a keyword _param such that 

put-implementation % ( _param = param_arg )

a returns a different (but not necessarily independent) implementation, based on the information in the right-hand-side.

Procedure

In each step below the user is required to write a piece of code within namespace boost::assign::v2. The notation typename..., here, is not a variadic template parameter. It means that the choice of template parameters are unspecified.

  1. Define 
    template<typename Arg>
struct param
{
	// Constructor
    implementation-defined get()const; 
};
  2. Define 
    struct param_keyword
{
	template<typename Arg>
    param<Arg> operator=(Arg const&); 
};
    where Arg is the type of param_arg.
  3. Define 
    namespace modifier_tag{

   	template<typename ...> struct param{};

}
  4. Define 
    namespace put_parameter{

   	template<typename Arg>
   	struct param{

           template<typename OldFun, typename OldTag>
           struct apply; // must derive from put_parameter::pair<>

   	};

}
  5. Specialize 
    namespace put_aux{

       template<typename...>
       class modifier<modifier_tag::param<Tag> >
       {
           public:

           modifier(){}
           explicit modifier(implementation-defined); 

           template<typename V, typename T>
           void impl(V& v, T& t)const; // puts t in v

           template<typename V, typename T>
           void impl(V& v, T* t)const; // for ptr_container
       };
   	
   }
    where implementation-defined is the result type of 
    ( _param = param_arg ).get()
  6. Expand 
    BOOST_ASSIGN_V2_PUT_MODIFIER_EXT_XXX(param)
