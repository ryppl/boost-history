Crtp

Header

boost/assign/v2/interpreter/crtp.hpp

Description

A crtp class that exposes a functor interface. Arguments are forwarded to a data-generator whose result is forwarded to a container modifier.

Synopsis 

namespace interpreter_aux{


    template<
        typename D              // Derived
        , typename C            // Container
        , typename F            // Data generator
        , typename MTag         // Modifier tag
        , typename DTag         // Data tag
    >
    class interpreter_crtp
    {

        public:

        typedef  interpreter_modifier<Tag> 
        modifier_type;

        interpreter_crtp(){}
        explicit interpreter_crtp( F const& f );
        interpreter_crtp( F const& f, modifier_type const& m );

        typedef D const& result_type;

        template<typename...Args>
        result_type operator()( Args&&...args )const;

        template<typename R>
        result_type for_each( R&& range )const;

        template<
            int I,
            typename R  // Range of tuples, each of size I
        >
        result_type for_each( R&& range )const;

    };

}// interpreter_aux

Header

boost/assign/v2/interpreter/csv.hpp

Description

Extends the interface of interpreter_crtp with a csv-syntax. The csv-form designates a syntax that is dual to a functor, that is charaterized by an arity parameter, I. One invocation of the csv-form with an argument list of size Ixn is equivalent to n invocations of the functor, each with I arguments. This form has two variants : basic (size equals one) and extended (size is specified by an integer template parameter).

Description

Interpreter's modifier. The value supplied by the data-generator is forwarded to the interpreter's modifier, whose job is to insert that value in a container.
