#define BOOST_ASSIGN_V2_LIMIT_CSV_ARITY unspecified
#define BOOST_ASSIGN_V2_LIMIT_LVALUE_CONST_ARITY unspecified

Semantics

Constant

Default

Description

BOOST_ASSIGN_V2_LIMIT_ARITY

BOOST_MPL_LIMIT_METAFUNCTION_ARITY

A functor's max # of arguments

BOOST_ASSIGN_V2_LIMIT_CSV_ARITY

20

Limits a csv-function's max # of arguments

BOOST_ASSIGN_V2_LIMIT_LVALUE_CONST_ARITY

2

A functor's max # of arguments for which any mixture of lvalue and const-lvalues is preserved [a]

[a] Beyond this limit, arguments are treated as either all lvalues or all const-lvalues.

Keyword

Header

boost/assign/v2/detail/support/keyword.hpp

Description

A keyword is a const object whose identifier has a leading underscore, that refers to an action or data-structure that has or is likely to have some use in other parts of the library, as part of a class specialization, function dispatch etc.

Synopsis 

namespace keyword_aux{

    struct ignore{    
        ignore(){} 
        template<typename Anything> ignore(Anything){}
    };
    struct element;
    struct key;
    struct map;
    struct nil;
    struct use_default;
    struct value;

}//keyword_aux
    typedef keyword_aux::ignore         ignore_;
    typedef keyword_aux::element        element_;
    typedef keyword_aux::key            key_;
    typedef keyword_aux::map            map_;
    typedef keyword_aux::nil            nil_;
    typedef keyword_aux::use_default    use_default_;
    typedef keyword_aux::value          value_;    

    const ignore_ _ignore;
    const element_ _element;
    const key_ _key;
    const map_ _map;
    const nil_ _nil;
    const use_default_ _use_default;
    const value_ _value;

Traits

Header

boost/assign/v2/support/traits.hpp

Description

Meta-functions that map a container to an associated type.

Synopsis 

template<typename C>
struct value_container_element;

template<typename C>
struct value_container_key;

template<typename C>
struct value_container_mapped;

template<typename C>
struct value_container_value;

Semantics

For value containers, the semantics each of meta-function is explicit from its name. For a pointer container, the latter is first mapped to its value-container counterpart, and the same meta-function evaluated over the latter.
