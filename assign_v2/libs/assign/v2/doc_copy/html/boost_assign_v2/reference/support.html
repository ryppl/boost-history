Description

Constants that control arity of functors and csv functions, throughout this library, under C++03 (not C++0x)

Synopsis 

/*
   Maximum arity for which functors are overloaded for any combination of const
   and non-const lvalues under C++03. Be warned that the number of function
   overloads increases geometrically with this number.
*/
#define BOOST_ASSIGN_V2_LIMIT_FUNCTOR_CONST_NON_CONST_ARITY 2


/* n = BOOST_ASSIGN_V2_LIMIT_FUNCTOR_ARITY
   Overload                           | Arity
   -------------------------------------
   functor()                          |    0
   functor(x[0])                      |    1
   functor(x[0], x[1])                |    2
   ...
   functor(x[0], ..., x[n-1])         |    n
*/

#define BOOST_ASSIGN_V2_LIMIT_FUNCTOR_ARITY BOOST_MPL_LIMIT_METAFUNCTION_ARITY



/* n = BOOST_ASSIGN_V2_CSV_LIMIT_ARITY
   Overload                           | Arity
   -------------------------------------
   csv_fun(t[0])                      |    1
   csv_fun(t[0], t[1])                |    2
   ...
   csv_fun(t[0], ...., t[n-1])        |    n
*/
#define BOOST_ASSIGN_V2_LIMIT_CSV_ARITY 12

Keyword

Header

boost/assign/v2/detail/support/keyword.hpp

Description

A keyword is a const object whose identifier has a leading underscore, that refers to an action or data-structure that has or is likely to have some use in other parts of the library, as part of a class specialization, function dispatch etc.

Synopsis 

struct kwd_ignore_
{
    kwd_ignore_(){}
    template<typename Anything> kwd_ignore_(Anything){}
};
struct kwd_element_;
struct kwd_key_;
struct kwd_map_;
struct kwd_nil_;
struct kwd_use_default_;
struct kwd_value_;


const kwd_ignore_ _ignore;
const kwd_element_ _element;
const kwd_key_ _key;
const kwd_map_ _map;
const kwd_nil_ _nil;
const kwd_use_default_ _use_default;
const kwd_value_ _value;

Traits

Header

boost/assign/v2/support/traits.hpp

Description

Meta-functions that map a container to an associated type.

Synopsis 

template<typename C>
struct value_container_element;

template<typename C>
struct value_container_key;

template<typename C>
struct value_container_mapped;

template<typename C>
struct value_container_value;

Semantics

For value containers, the semantics each of meta-function is explicit from its name. For a pointer container, the latter is first mapped to its value-container counterpart, and the same meta-function evaluated over the latter.
