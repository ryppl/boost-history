Organization

Headers

This is a header only template library:

  • The directory containing Boost has to be in the compiler's search path
  • Using a given feature of Boost.Assign 2.0 only requires including the corresponding header file.

It is advised to include headers selectively, in general, but particularly using C++03 which compiles more slowly [2]. Headers can be looked for

  • by module, in directory boost/assign/v2
  • by identifier, in directory boost/assign/v2/include, [3]

Accordingly, this statement : 

#include <boost/assign/v2.hpp>

includes all the modules [4], and is equivalent to defining all identifiers, as follows: 

#include <boost/assign/v2/include.hpp>

The functionality of Boost.Assign (1.0)[5] does not clash with that versioned 2.0, although it is recommended, for coherence, to use one, but not both, in a given project. The following statement: 

#include <boost/assign.hpp>

retrieves the content of version 1.0, not 2.0.

Namespaces

The namespace organization is summed up below: 

namespace boost{
namespace assign{

    // Boost.Assign (1.0) identifiers

namespace v2{

    // Boost.Assign 2.0 identifiers

namespace ref{

    // Features pertaining to references
	
}// ref
}// assign
}// v2
}// boost


[2] Due to heavy overloads. As an indication, compiling all of libs/assign/v2/test takes about 6 mn using C++03, and 1 mn using C++0x, under the machine specifaction given here.

[3] Excepted boost/assign/v2/include/option.hpp, which includes all the options. See section Option for individual headers.

[4] A few headers are not included in that of their enclosing module. Where applicable, this will be mentioned

[5] The presence of parentheses is to mark the fact that this version number is retrospective : Boost.Assign and Boost.Assign (1.0) designate the same library
