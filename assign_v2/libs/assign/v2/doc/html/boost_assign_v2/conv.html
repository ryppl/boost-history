Convention

Here are conventions used in this documentation. They are hopefully intuitive so the impatient reader can skip them.

Table 1.1. Symbol

Symbol

Description

cont

A container

t

A value

arg

Arguments to a function


Identifier T represents the type of t, and C that of cont.

Table 1.2. Index

Expression

Description

i = 0, ..., n-1

Indexes a sequence of t's

j = 0, ..., k-1

Indexes arguments of a k-ary function


Table 1.3. Bracket

Expression

Description

t[i]

ith value

arg[j]

jth argument at unspecified i

arg[i,j]

jth argument at i


Table 1.4. Alias

Expression

Description

x, y, z

t[0] , t[1] , t[2]


In the example involving a calendar, n = 3 and k = 2 for i = 0, 1 ,2, corresponding the months of January, February and March, respectively. In the example involving a stable partition, n = 6 and k = 1.

Code that describes a feature such as under heading Synopsis is expressed in terms of the preceding symbols and indexes, never actual data-types (such as int). Identifiers in the namespace of this library, boost::assign::v2, are not fully qualified. Another way to say this, is to assume that the code is preceded by the following using declaration. 

using namespace boost::assign::v2;

Any code snippet under heading Example is assumed to be preceded by the include statement under heading Header appearing in the nearest enclosing section (if not the current one, that of the top level etc.). Furthermore, the following aliases are used: 

namespace as2 = boost::assign::v2;
namespace lambda = boost::lambda;

Headers or definitions that are not part of this library are not explicitly included in the documentation. They can be found in the sources files of the test files located in this directory .
