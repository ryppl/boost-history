Tutorial

Requisites

We assume in this tutorial that all the header files are included, as shown in section Organization and 

using namespace boost::assign::v2;

Headers from other libraries will not be shown in the code snippets of this tutorial, but they can be found in the tutorial's actual source file.

Automatic behavior

We refer, here, to the introduction's example and recall that the interpreter returned by put is variadic and

  • that its arguments are forwarded to a data-generator. See here for further detail.
  • determines an appropriate modifier, insert in case of a map, based on a set of rules, such that the interface is uniform across different container categories.

Csv syntax

Consider this example: 

std::vector<int> numeric( 10 ); iota( numeric, 0 ); 
typedef std::string str_; typedef variant< int, str_ > data_; 
array<data_, 16> keypad;

csv_put( keypad, "+", "-", "*", "/", "=", "." , as_arg_list( numeric ) );

assert( get<str_>( keypad.front() ) == "+" );
assert( get<int>( keypad.back()  ) == 9 );

Equivalent to put( keypad )( "+" )( "-" )...( "." )( 0 )...( 9 )

Prefix csv[4] indicates an alternative syntax that is dual to a functor, as can be garnered from the callout. Other names starting with csv are treated similarly.

Another thing to notice is that the last argument (in this example) is a range, numeric, with a special wrapper around it, as_arg_list. The latter instructs the interpreter to scan the range's elements, and process each as if passed as an individual argument.

Pipe syntax

Suppose we wish to put a set of integers in a container and, in the same statement, pass it through a mutating algorithm: 

std::deque<int> source( 6 ), target; iota( source, 0 );

range::stable_partition( 
    target | delay_csv_put( source ), 
    lambda::_1 % 2 
);

The vertical bar, |, expresses the on-the-fly modification we were looking for, and call it piping after the related idiom by the same name. A related feature is Boost.Range's range adaptors.

Container generation

After having assigned target and, in the same statement, passed it through a stable partition algorithm, let's verify the state of the container against that of an in place generated container: 

assert(    
    range::equal( target, csv_deque(1, 3, 5, 0, 2, 4) ) 
);

In keeping with our previously established csv convention, container generating function csv_deque is paired with a functor form, deque. Another convention, that we now introduce by way of example, is that which governs the generation of an empty container: 

assert( deque<int>( _nil ).size() == 0 );    // Functor form
assert( csv_deque<int>().size() == 0 );      // Csv form

Pointer-containers

To insert elements in a pointer container, pretend it is a value-container: 

typedef std::string word_; ptr_set<word_> words;
csv_put( words, "isomer", "ephemeral", "prosaic" );

assert( words.count( "isomer" ) == 1 );
assert( words.count( "prosaic" ) == 1 );

Calls assoc.insert( new word_( s ) ) for s = "isomer", "ephemeral", "prosaic"

Options

The automatically deduced data-generator can be overriden, such as follows: 

std::vector<double> exponent;

typedef function<double(double)> f_;
csv_put(
    exponent
    , _option % ( _data = f_(  log10  ) )
    , 1.0, 10.0, 100.0, 1000.0, 10000.0
);

double eps = numeric::bounds<double>::smallest();
assert( fabs( exponent.front() - 0.0 ) < eps );
assert( fabs( exponent.back() - 4.0 ) < eps );

Equivalent to ( put( exponent ) % ( _data = f_( log10 ) ) )( 1.0 ) ...( 10000.0 )

Similarly, the modifier can be explicitly specified, such as follows: 

boost::circular_buffer<int> cb( 3 ); 

assert(
    range::equal(
        cb | delay_csv_put( _push_front, csv_deque( 3, 2, 1 ) ), 
        csv_deque( 1, 2, 3 ) 
    )
);

These are basic examples, each involving one option. Notice that only the form csv_put requires the use of keyword _option. See section Option for more advanced options. Composing m options is achieved as follows : option1 % ... % optionm. Finally, extending the library with a new modifier-option is fairly easy, thanks to macros that manage the overhead.

Reference arrays

In the examples that precede, ref::csv_array, which returns a reference-array, could be substituted for csv_deque. The reciprocal is not true, in general. A reference-array is preferable if the values are expensive to copy, and it is necessary for working specifically with references, such as as follows: 

int x = 4, y = 6, z = -1;
int const& max = *max_element( ref::csv_array( x, y, z ) );

assert( &max == &y );

In keeping with our csv convention, ref::csv_array has a functor dual, ref::array. Unlike put or deque, it is unary (not variadic).

Chaining

Chaining ranges is already provided by join in Boost.Range. Here's a variant that caters to reference arrays: 

std::vector<int> source( 8 ); boost::iota( source, 1 );

array<int, 4> head; int t, a, i, l;
boost::copy(
    source,
    boost::begin(
        head | _chain( 
            ref::csv_array( t, a, i, l ) | ref::_get 
        )
    )
);

assert( range::equal( head, csv_deque( 1, 2, 3, 4 ) ) );
assert( t == 5 ); 
assert( a == 6 ); 
assert( i == 7 ); 
assert( l == 8 );

It is noteworthy that this algorithm modifies x, y and z even though the array returned by ref::csv_array is an rvalue. The part of the expression involving ref::_get tells the implementation that we wish to use the reference-array in write mode i.e. modify the values of its held references [5].

Conversion

This section deals with conversion from a Range to collections of elements that need not support iterators. For example, 

typedef std::queue<int> C;

C fifo = converter( csv_deque( 1, 2, 3 ) );

assert( fifo.front() == 1 ); 
assert( fifo.back() == 3 );

Notice, indeed, that the left hand side of = is an adapter so, unlike the container in the right hand side, it is not a Range. The conversion, above, is implicit, and is made possible by a conversion operator fitted to the object returned by converter. Sometimes, explicit conversion is unavoidable and, continuing the above snippet, is illustrated as follows: 

assert(
    ( csv_deque( 1, 1, 3 ) | convert<C>() ) < fifo
);


[4] csv stands for comma separated values

[5] In read mode, the context usually suffices, to ensure conversion from each of the reference-array's element (a reference wrapper) to the reference it holds
