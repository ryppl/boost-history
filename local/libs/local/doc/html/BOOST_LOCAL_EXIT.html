Macro BOOST_LOCAL_EXIT

BOOST_LOCAL_EXIT — This macro starts the declaration of a local exit.

Synopsis

// In header: <boost/local/exit.hpp>

BOOST_LOCAL_EXIT(binding_list)

Description

This macro must be used within a declarative context, it must be followed by the local exit body code and then by the BOOST_LOCAL_EXIT_END macro (see the Tutorial section): 

  { // Some declarative context.
      ...
      BOOST_LOCAL_EXIT(bindings) {
          ... // Block code.
      } BOOST_LOCAL_EXIT_END
      ...
  }

As usual, exceptions specifications can be optionally programmed before the body code block (but the specifications will only apply the body code and not to the library code automatically generated by the macro expansion, see Advanced Topics).

Within templates, the special macro BOOST_LOCAL_EXIT_TPL must be used instead of BOOST_LOCAL_EXIT.

Note: A return; instruction from within a local exit body jumps to the end of the local exit body and it does not return the enclosing scope.

Parameters:

bindings On all C++ compilers, the sequencing macro syntax defined by the following grammar is used to specify the variables in scope to bind: 
      bindings:
              void_list | bound_list
      void_list:
              (void)
      bound_list:
              (bound_parameter1) (bound_parameter2) ...
      bound_parameter:
              [const] bind[&] variable_name
Where the following lexical conventions have been used: token1 | token2 means either token1 or token2; [token] means either token or nothing (i.e., token is optional).On C99 and later compilers which support variadic macros, the above grammar can be modified as follow to define the variadic macro syntax that can also be used to specify the variables in scope to bind: 
      void_list:
              void
      bound_list:
              bound_parameter1, bound_parameter2, ...
Note that the variadic macro syntax uses commas , instead of parenthesis () to separate the parameters and therefore resembles the usual C++ function parameter declaration syntax more closely than the sequencing macro syntax. However, the variadic macro syntax will only work on C++ compilers that support variadic macros so it should be used with care in order to avoid portability issues.Finally, on C++ compilers which support empty macro parameters, the above grammars can be modified as follow to define the empty macro syntax that can can also be used to specify an empty list of variables to bind: 
      void_list:
              ()
Note that the empty macro parameter syntax uses the usual C++ syntax to specify empty bound variable list but it will only work on compilers that support empty macro parameters so it should be used with care in order to avoid portability issues.

Note: This macro cannot be used multiple times on the same line because it internally uses the line number __LINE__ to generate unique identifiers.
