Struct template function_ref

boost::local::function_ref — Template to hold a reference to a local function while supporting eventual default function parameters.

Synopsis

// In header: <boost/local/function_ref.hpp>

template<typename F, size_t defaults = 0> 
struct function_ref {
};

Description

This template defines several specializations to handle a generic number N of function parameters some of which can have default values. The number of supported function parameters goes from 0 (for a function with no parameter) to a maximum of BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX.

Each template specialization defines call operators operator()(...) with a different set of parameters to handle the number of default parameters specified by defaults (see Advanced section): 

  template<typename Result, typename Arg1, ..., typename ArgN, size_t defaults = 0>
  class function_ref<Result (Arg1, ..., ArgN), defaults> {
  public:
      Result operator()(Arg1, ..., ArgN-2, ArgN-1, ArgN);
      // Call operators to handle default parameters:
      Result operator()(Arg1, ..., ArgN-2, ArgN-1);       // iff defaults >= 1
      Result operator()(Arg1, ..., ArgN-2);               // iff defaults >= 2
      ...                                                 // etc

      // Copy constructor and assignment operator for local functions:
      function_ref(local_function<F, defaults>& ref);
      function_ref& operator=(local_function<F, defaults>& ref);
  };

Where:

  • Result is the function result type. It can be void.

  • ArgN is the last function parameter type, ArgN-1 is the second last function parameter type, etc. These are all optional (none is specified for a function with no parameter, only Arg1 is specified for a function with only one parameter, etc).

  • The operator Result operator()(Arg1, ..., ArgN-2, ArgN-1) is defined if and only if there are one or more default parameters (defaults >= 1), the operator Result operator()(Arg1, ..., ArgN-2) is defined if and only if there are two or more default parameters (defaults >= 2), etc.

  • local_function<F, defaults> is an internal type for a local function with a signature matching F and with a number of default parameters equal to defaults.

  • The copy constructor and assignment operator operator= allow to assign this template to a reference to a local function with a signature matching F and with a number of default parameters equal to defaults.

Warning: Programmers must make sure that the local function survives the scope of the function reference (otherwise the reference will be invalid and its use will generate a run-time error as usual with C++ references).

Parameters:

F The function signature expressed using the Boost.Function's preferred syntax: F = Result (Arg1, ..., ArgN).
defaults The number of the function default parameters in [0, BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX]. As usual in C++, default parameters are counted starting from the last parameter: defaults = 0 means that no parameter is optional, defaults = 1 means that the last parameter is optional, defaults = 2 means that the last two parameters are optional, etc.

Note: This template is similar to boost::function<> but it also supports eventual default parameters.

See: Advanced section, BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX, BOOST_LOCAL_FUNCTION, Boost.Function.
