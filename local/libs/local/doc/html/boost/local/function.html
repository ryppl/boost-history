Struct template function

boost::local::function — Functor with support for default function parameters.

Synopsis

// In header: <boost/local/function.hpp>

template<typename F, size_t defaults = 0> 
struct function {
};

Description

This template defines several specializations to handle a generic number of function parameters some of which can have default values. The number of supported function parameters N goes from 0 (for a function with no parameter) to a maximum of BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX.

Each template specialization defines call operators operator()(...) with a different set of parameters to handle the number of default parameters specified by defaults (see Advanced Topics section): 

  template<typename Result, typename Arg1, ..., typename ArgN, size_t defaults = 0>
  class function<Result (Arg1, ..., ArgN), defaults> {
  public:
      Result operator()(Arg1, ..., ArgN-2, ArgN-1, ArgN);
      // Call operators to handle default parameters:
      Result operator()(Arg1, ..., ArgN-2, ArgN-1); // iff defaults >= 1
      Result operator()(Arg1, ..., ArgN-2);         // iff defaults >= 2
      ...                                           // etc

      // Default and copy constructors, and assignment operator.
      function();
      function(local_function_type<F, defaults> local_function);
      function& operator=(local_function_type<F, defaults> local_function);
  };

Where:

  • Result is the function result type. It can be void.

  • ArgN is the last function parameter type, ArgN-1 is the second last function parameter type, etc. These are all optional: None is specified for a function with no parameter; Only Arg1 is specified for a function with only one parameter; Etc.

  • The operator Result operator()(Arg1, ..., ArgN-2, ArgN-1) is declared if and only if there are one or more default parameters (defaults >= 1), the operator Result operator()(Arg1, ..., ArgN-2) is defined if and only if there are two or more default parameters (defaults >= 2), etc.

  • local_function_type<F, defaults> is an internal (i.e., implementation specific) type for a local function with a signature matching F and with a number of default parameters equal to defaults.

  • The default constructor allows to constructor this functor without assigning it to any function. A run-time error is generated if the call operator operator() is invoked for a functor not assigned to any function.

  • The copy constructor and assignment operator operator= allow to assign this functor to a local function with a signature matching F and with a number of default parameters equal to defaults.

Warning: Programmers must make sure that the local function assigned to this functor survives the scope of the functor (otherwise the functor will be invalid and its use will generate a run-time error as usual with C++ functors).

Parameters:

F The function type specifying function signature expressed using the same syntax as Boost.Function's preferred syntax: F = Result (Arg1, ..., ArgN).
defaults The number of the function default parameters in [0, BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX]. As usual in C++, default parameters are counted starting from the last parameter: defaults = 0 means that no parameter is optional, defaults = 1 means that the last parameter is optional, defaults = 2 means that the last two parameters are optional, etc.

Note: This functor is similar to Boost.Funciton's boost::function<> but it also supports eventual default parameters.

See: Advanced Topics section, BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX, BOOST_LOCAL_FUNCTION_PARAMS, BOOST_LOCAL_FUNCTION_NAME, Boost.Function.
