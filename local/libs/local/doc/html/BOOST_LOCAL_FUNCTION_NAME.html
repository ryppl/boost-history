Macro BOOST_LOCAL_FUNCTION_NAME

BOOST_LOCAL_FUNCTION_NAME — This macro is used to specify the local function name.

Synopsis

// In header: <boost/local/function.hpp>

BOOST_LOCAL_FUNCTION_NAME(name)

Description

This macro must follow the local function body code block { ... } as shown in the BOOST_LOCAL_FUNCTION_PARAMS documentation.

The local function name can be prefixed by the "keyword" inline: 

  ... BOOST_LOCAL_FUNCTION_NAME(inline name)

This increases the chances that the compiler will be able to inline the local function calls (thus reducing their run-time). However, inlined local functions cannot be passed as template parameters (on ISO C++ compliant compilers, see BOOST_LOCAL_CONFIG_COMPLIANT, but they can be passed as template parameters on C++0x compilers). Furthermore, inlined local functions cannot be assigned to other functors like boost::function. On C++0x compilers, there is no need to declare a local function lined because this library will automatically use C++0x specific features to inline the local function while always allowing to pass it as a template parameter.

The local function name can also be prefixed by the "keyword" recursive: 

  ... BOOST_LOCAL_FUNCTION_NAME(recursive name)

This allows the local function to recursively call itself from its body (as usual in C++). However, compilers have not been observed to be able to inline recursive local function calls (not even when the recursive local function is also declared inlined).

Parameters:

name The name of the local function. The local function name can be prefixed by the "keyword" inline to declare the local function inlined or by the "keyword" recursive to declare the local function recursive (see the Advanced Topics section). The local function name cannot be the name of an operator operator... and it cannot be the same name of another local function declared within the same enclosing scope (i.e., it is not possible to overload local function declarations, see the Advanced Topics section).

See: BOOST_LOCAL_FUNCTION_PARAMS, Tutorial section, Advanced Topics section.
