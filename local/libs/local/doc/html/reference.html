Header <boost/local/block.hpp>

Local blocks allow to program code that is executed accessing some of the variables in scope as constants. 


BOOST_LOCAL_BLOCK(parenthesized_binding)
BOOST_LOCAL_BLOCK_TPL(parenthesized_binding)
BOOST_LOCAL_BLOCK_END

Header <boost/local/config.hpp>

Configuration macros allow to change the behaviour of this library at compile-time. 


BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX
BOOST_LOCAL_CONFIG_THIS_PARAM_NAME

Header <boost/local/exit.hpp>

Local exits allow to program code that is executed when the enclosing scope is exited. 


BOOST_LOCAL_EXIT(parenthesized_binding)
BOOST_LOCAL_EXIT_TPL(parenthesized_binding)
BOOST_LOCAL_EXIT_END

Header <boost/local/function.hpp>

Local functions allow to program functions locally within the scope where they are needed. 


BOOST_LOCAL_FUNCTION(parenthesized_signature)
BOOST_LOCAL_FUNCTION_TPL(parenthesized_signature)
BOOST_LOCAL_FUNCTION_END(local_function_name)
BOOST_LOCAL_FUNCTION_END_RENAME(new_local_function_name)

Header <boost/local/function_ref.hpp>

Local function references can be passed as function and template parameters, they can be assigned to variables, etc. 

namespace boost {
  namespace local {
    template<typename F, size_t defaults = 0> struct function_ref;
  }
}

Header <boost/utility/identity.hpp>

Identity macros are used to pass expressions with commas (not already wrapped within round parenthesis) as macros parameters.

Note: The only way (at least as far as the authors know) to pass an arbitrary number of commas within macro parameters without explicitly indicating the number of commas is to wrap the parameter within round parenthesis that are not replaced by macro expansion and then to remove the extra parenthesis at compile-time (for type expressions) or at run-time (for value expressions). This is what these macros do. 


BOOST_IDENTITY_TYPE(parenthesized_type)
BOOST_IDENTITY_VALUE(parenthesized_value)
