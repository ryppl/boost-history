Execute as sub-task

boost::task::as_sub_task is a convenient way to execute a sub-task. If the parent task is executed inside a thread-pool the sub-task is put into the local-queue of the worker-thread in the other case the sub-task will be executed in a new thread. boost::task::as_sub_task is used as default execution-policy for boost::task::async(), 

boost::task::task< long > t( fibonacci, 10);

boost::task::handle< long > h(
	boost::task::async(
		boost::move( t),
		boost::task::as_sub_task() ) );
