Utilities

Boost.Task provides some free-functions which can be used inside a boost::task::task to access some informations (like if the boost::task::task runs in a thread-pool).

reschedule_until

In the function boost::this_task::reschedule_until( Pred const&) allows to synchronize the task with other asynchronous events without blocking the worker-threads (bool Pred::operator()() must not block). The current task will be rescheduled until the passed predicate becomes true.

get_pool

The pool in which the current code (boost::task::task) is executed can be accessed via boost::this_task::get_pool(). If hte code is not executed by a worker-thread an assertion is raised.

runs_in_pool

In order to check if the current code is executed in a thread-pool boost::this_task::runs_in_pool() should be used.

worker_id

boost::this_task::worker_id() returns the boost::thread::id of the worker-thread executing the current boost::task::task.

delay

The execution of a boost::task::task can be delayed for a time-duration or until a specific time-point with boost::this_task::delay().

yield

If a boost::task::task detects that it would bould block it can yield itself with boost::this_task::yield() so that the worker-thread can execute another boost::task::task in the meantime.

interrupt

A boost::task::task can interrupt itself via boost::this_task::interrupt().
