// Create a new Erdos-Renyi past-the-end edge iterator

erdos_renyi_random_iterator<SizeType, Probability, RandomGenerator>
make_erdos_renyi_random_iterator(SizeType n, Probability, RandomGenerator&);

erdos_renyi_random_iterator is an iterator that enumerates edges generated using the Erdos-Renyi random graph model [58]. This model takes two parameters, n and p, where n is the number of vertices in the graph and p is the probability of there being an edge between any two randomly-selected vertices in the graph.

The Erdos-Renyi random graph iterators are intended for use in the construction of graphs. The generated edges are pairs of indices, as used by the adjacency_list and adjacency_matrix constructors. They must be provided with the number of vertices n, the probability p, a random number generator (such as minstd_rand), and the directed_category of the graph (either directed_tag or undirected_tag). The final boolean value states whether the generator may generate self edges (where the source and target refer to the same vertex). The past-the-end iterator denoting the end of the sequence of random edges is generated by omitting the directed tag and self_edges iterator.

Class template erdos_renyi_random_iterator

template<typename SizeType, typename Probability, typename RandomGenerator = minstd_rand>
  class erdos_renyi_random_iterator
  {
  public:
    // Create a new Erdos-Renyi edge iterator
    erdos_renyi_random_iterator(SizeType n, Probability p, 
                                RandomGenerator& gen, bool directed,
                                bool self_edges = false);
 
    // Create a new Erdos-Renyi past-the-end edge iterator
    erdos_renyi_random_iterator(SizeType n);
  };

Class template erdos_renyi_random_iterator models the Input Iterator concept. Its constructors accept the same parameters as the make_erdos_renyi_random_iterator functions, with two exceptions: (1) The first constructor accepts the directed parameter as a (run-time) boolean value instead of as a directedness tag type, and (2) the second constructor accepts only the number of vertices in the graph, because the probability and random number generator are unused.
