Installing LUID

Getting Boost.LUID

You can get LUID by downloading luid.zip from http://www.boost-consulting.com/vault/index.php?directory=Data%20Types

Building Boost.LUID

There is no need to compile Boost.LUID, since it's a header only library. Just include your Boost header directory in your compiler include path.

Boost.LUID depends on Boost. You must use either Boost version 1.35.x or the version in SVN trunk. In particular, Boost.LUID depends on:

Boost.DateTime

which needs separate compilation. However, the subset used by Boost.LUID does not need any separate compilation so the user can define BOOST_DATE_TIME_NO_LIB to avoid Boost from trying to automatically link the Boost.DateTime.

Boost.DynamicBitSet

to store the recovered luids

Boost.Integer

to obtain the default bounds

Boost.Interprocess

to lock on multi process scope and to store on shared memory or mapped memmory files when the lifetime is longer than the process one.

Boost.Intrusive

to store the recovered luids

Boost.MPL

for all the metaprograming task

Boost.Parameters

to pass the constructor parameters and to configure the template parameters

Boost.Threads

to lock on multi_threaded scope.

Boost.Threads

needs separate compilation. However, you will need only to link with depending on the features used.

Boost.TypeTraits
