This section relies on notational conventions laid out in section Notation. Examples of this section assume preceding code laid out in section Examples' background.

Header 

#include <boost/assign/v2/anon.hpp>

Description

A pair of functions, csv_anon() and anon(), to construct in place (hence the name anonymous) a container of values that is akin to std::deque<>.

csv_anon

[csv_anon]

Argument forwarding and the ability to specify a user-specified implementation (functor or modifier) are two features provided that we encountered in the put framework, specifically sections Standard modifier and Argument forwarding, and which also are provided here.

Expression

Result type

anon<T>(_nil)( arg[0][0], ..., arg[0][k[0]])...( arg[n-1][0], ..., arg[n-1][k[n-1]])

typename result_of::anon<T>::type


Notice that the result type of csv_anon<T>() and anon<T>() are of the same type. Only using the former can T be deduced from the arguments[6], and only using the latter can an empty container, by passing keyword _nil, be created.

Example

[anon]


[6] If, in the expression for argument forwarding, k[0] = 1 and it is known that arg[0][0] is of type T, such that we may label it, instead, t[0], the first two calls to operator() are equivalent anon<T>(_nil)( t[0] ). Under these conditions, it is equivalent, but preferable to call csv_anon( t[0] ).
