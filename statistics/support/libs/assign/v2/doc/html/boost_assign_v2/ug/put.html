operator|()

_csv_put
_put
Pointer container
put_range
put_convert
Header 

#include <boost/assign/v2/put/operator.hpp>

Description

The semantics of operator|(), in this library, emulate those of range adaptors to the extent that they are composable and therefore convenient in conjunction with range algorithms. In the context of range adaptors, expression 

lhs | rhs

maps a range, identified by lhs, according to specifications encapsulated by rhs. In the remainder of this section, unless otherwise stated, the above expression returns a reference to container lhs, after modifying its state according to specifications embodied by rhs.

_csv_put
Header 

#include <boost/assign/v2/put/operator/csv.hpp>

Synopsis

Expression

Result

Side effect

cont | _csv_put( t[0], ....,t[n-1] )

A reference to cont

Puts t[0], ....,t[n-1] in cont

Comments

The particular side effect is specified in section . For common needs, it is that which the user expects. For example, if cont is of type std::vector<>, the expression forwards to cont.push_back(). In the right hand side of operator|, the result of _csv_put() internally keeps references to its arguments so as to avoid any storage overhead.

Example 

std::vector<int> cont;
boost::for_each(
    cont | as2::_csv_put( -1, 0, 1 ),
    os << bl::_1 << ' '
);
//outputs -1 0 1

_put
Header 

#include <boost/assign/v2/put/operator/functor.hpp>

Introduction

Consider this example 

typedef std::map<const char*, int> cont_;
typedef cont_::value_type type;
cont_ cont;
BOOST_ASSERT(
    (
        cont | as2::_csv_put(
            type("x", -1),
            type("y", 0),
            type("z", 1)
        )
    )["y"] == 0
);

Instead of explicitly calling type() for each set of arguments, what we'd like, the topic of this section, is to automatically forward to the constructor.

Synopsis

In terms of our notation, for n = 1,

Expression

Result

Side effect

cont | _put(arg[0], ....,arg[k])

A reference to cont

Forwards to T(arg[0], ....,arg[k]) and puts the result in the container.

The appropriate type, T, is deduced at compile time, from the type of destination container, cont. The above extends to arbitrary n as follows: 

cont | _put(arg[0][0], ....,arg[0][k[0]])...(arg[n-1][0], ....,arg[n-1][k[n-1]])

Example 

std::map<const char*, int> cont;
BOOST_ASSERT(
    (
        cont | as2::_put("z", 1)("x", -1)("y", 0)
    )["y"] == 0
);

Pointer container

This section discusses the side effect of operator|(), with a pointer container as left-argument. Consider this example: 

boost::ptr_vector<int> cont;
boost::for_each(
    cont | as2::_csv_put( -1, 0, 1 ),
    os << bl::_1 << ' '
);
// output -1 0 1

Copies of x, y and z are allocated on the heap, and their pointers put into cont, using push_back(), in this case.

put_range
Header 

#include <boost/assign/v2/put/operator/range.hpp>

Synopsis

Let t[0]...t[n] denote the dereferenced values of range r.

Expression

Result

Equivalent expression 

cont | put_range( r );

A reference to cont 

cont | _put(t[0])...(t[n])

Caution

Whenever possible, boost::copy(r, boost::begin(cont) ) is probably preferable, which tends to restrict the applicability of put_range to the case in which cont is an adapter.

Example 

std::vector<int> from( 3 );
from[0] = -1; from[1] = 0; from[2] = 1;
std::queue<int> to; // FIFO
BOOST_ASSERT(
    (
        to | as2::adaptor::_put_range( from )
    ).front() == -1
);
BOOST_ASSERT( to.back() == 1 );

operator%()

_fun
Modifier
Description

The put framework relies on traits of the container to be modified to determine the appropriate implementation. This has the advantage that the same and compact interface works across different container categories, whether Sequence, Associative or adapters.

To allow the user to override the implementation, however, this library provides a composable parameter passing mechanism. A parameter is denoted _p.

Synopsis

Expression

Result

implementation-defined % _p

An implementation whith a new state or (not xor) type

_fun
Header
Synopsis

See section passing a parameter.

Expression

Result

lhs % (_fun = f)

Replaces the internal functor of lhs by f

where lhs is either _csv_put or _put.

Example 

std::vector<int> cont;
boost::for_each(
    cont | (
        as2::_csv_put % (
            as2::_fun = (2 * bl::_1)
        )
    )( -1, 0, 1 ),
    os << bl::_1 << ' '
);
// outputs -2 0 2

Modifier
Header 

#include <boost/assign/v2/put/modifier.hpp>

Description

This section documents the mechanisms by which a passed value, t, is put in a container. These mechanisms are of two kinds, standard, and extended, the topic of the next sub-sections.

Standard
Header 

#include <boost/assign/v2/put/modifier/std.hpp>

Definition

A standard modifier is an implementation that, given a passed value, t, inserts it in _cont_` by calling one of its member function.

User specified

Rather than leaving the library to automatically deduce a standard modifier at compile time, the user can specify it explicitly by passing a keyword parameter, _p, with operator%.

Parameters

Here's a catalog of the keywords that control standard modifiers, and the container categories under which they are the default.

Keyword

Implementation

Default

_insert

cont.insert( t )

Associative

_iterate [a]

cont[ i++ ] = t

Static arrays

_push

cont.push( t )

Adapters

_push_back

cont.push_back( t )

Sequence

_push_front

cont.push_front( t )

[a] This keyword does not agree with our definition of a standard modifier which requires that the element be inserted, not merely assigned. However, it make sense from a user standpoint to leave that keyword in this table. Also see Extended functionality.

where index i is initialized to zero.

Example

The default standard modifier of std::list<> is push_back(). In this example, it is changed to push_front(): 

std::list<int> cont;
boost::for_each(
    cont | (
        as2::_csv_put % as2::_push_front
    )( -1, 0, 1 ),
    os << bl::_1 << ' '
);
// output 1 0 -1

Make sure to keep the statement involving operator% in between brackets, as shown above.

Customization
Header 

#include <boost/assign/v2/put/modifier/xxx.hpp>

Usage

Suppose the user has defined his/her own container and that baz() is a member function for putting an element in it: 

struct bar{ void baz(int); };

Expanding 

BOOST_ASSIGN_V2_PUT_MODIFIER_STD_XXX(baz)

creates a keyword _baz, such that the following statement 

bar cont;
cont | (_csv_put % _baz)( x, y, z );

forwards each of x, y, and z to cont.baz().
