where ref::assign_xxx is either of ref::assign_copy or ref::assign_rebind, and U is T const if t[i] is const or an rvalue, for at least one i, and U is T otherwise.

Assign copy

To work with references, the relevant namespace is ref, but we also need a nested namespace that specifies an assignment policy. For example, 

namespace ns = ref::assign_copy;
boost::for_each( ns::csv_anon( x, y, z ), printer(std::cout) );
//outputs (1,0)(NULL,0)(1,5)

The result from ns::csv_anon() is more than just a range. It has much of the functionality of the static array, boost::array<>, such as assign(), as we now illustrate, 

T x1, y1, z1;
ns::csv_anon( x1, y1, z1 ).assign( T( b, a ) );
BOOST_ASSERT( y1 == T( b, a ) );

Assign rebind

We now turn to the rebind assignment policy. Here, it is necessary to save the result: 

namespace ns = ref::assign_rebind;
typedef ns::nth_result_of::csv_anon<3, T const>::type result_;
result_ result = ns::csv_anon( x, y, z );
BOOST_ASSERT( &result[ 1 ] == &y );

Thus far, the side effect would be the same under the assign_copy policy. The effect of assigning a new object to each element of our result, however, is different: 

T w = T( b, a ); result.assign( w );
BOOST_ASSERT( &result[ 1 ] == &w );

The functor interface

The functor interface works as follows: 

namespace ns = ref::assign_rebind;
typedef ns::nth_result_of::anon<2, T const>::type result1_;
result1_ result1 = ns::anon( x )( y );

and result1 has all the functionality of that resulting from csv_anon(x, y). Unlike the latter, it's possible to append an element: 

typedef ns::nth_result_of::anon<3, T const>::type result2_;
result2_ result2 = result1( z );

Internally, result2 keeps a reference to result1, and not until the boost::array<> interface is called, as in 

boost::for_each( result2, printer(std::cout) );

is a static array allocated, with reference wrappers, in this case, for x, y and z,
