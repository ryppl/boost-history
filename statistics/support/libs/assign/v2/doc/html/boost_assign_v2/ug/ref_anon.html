Anonymous container of references

csv_anon
anon
Assignment
Namespace assign_copy
Namespace assign_rebind
Header 

#include <boost/assign/v2/ref.hpp>

Description

A pair of functions, csv_anon() and anon(), to construct in place (hence the name anonymous) a static array of references. In access mode, the latter can be used practically as boost::array<>. In write mode, since the underlying data is of reference type, the semantics of operator= and assign() are particular, which is the topic of section Assignment.

Notation

To proceed to the next sections, suffice it to know that csv_anon() and anon() are defined in both namespace assign_copy and assign_rebind, each corresponding to a particular assignment policy. Accordingly, we will use the following convention 

namespace ns = ref::assign_xxx;

where assign_xxx designates either of the two aforementioned namespaces.

Template argument U is, in terms of our notation, either of T or T const.

csv_anon

Synopsis

Expression

Result

ns::csv_anon<U>( _nil )

An object of type typename ns::nth_result_of::csv_anon<0, U>::type

ns::csv_anon( t[0], ..., t[n-1] )

An object of type typename ns::nth_result_of::csv_anon<n, U>::type

In the second expression, U == T const if and only if t[i] is passed as a const lvalue or an rvalue, for at least one i, and U == __value_type otherwise [3] .

anon

Synopsis

Expression

Result type

ref::assign_xxx::anon( _nil )

typename ref::assign_xxx::nth_result_of::anon<0, U>::type

ref::assign_xxx::anon( t[0]) ...(t[n-1] )

typename ref::assign_xxx::nth_result_of::csv_anon<n, U>::type

The determination of U for the second expression is identical to that described for csv_anon().

Example

The advantage over csv_anon() is that a result can be saved, 

namespace ns = ref::assign_rebind;
typedef ns::nth_result_of::anon<2, T const>::type result1_;
result1_ result1 = ns::anon( x )( y );

and later appended: 

typedef ns::nth_result_of::anon<3, T const>::type result2_;
result2_ result2 = result1( z );

but not until the boost::array<> interface is called, is a static array of references internally allocated: 

boost::for_each( result2, printer(os) );

In this example, the begin() and end() are the members of the interface that were presumably called by boost::for_each().

Throughout this section, the synopses assume 

typedef ref::assign_xxx::nth_result_of::csv_anon<n, T> Cont;

where assign_xxx is the namespace under consideration, and cont, of the type above, has been initialized.

Namespace assign_copy
Synopsis

Expression

Post condition 

cont[i] = t


cont[i] == t

Example

In access mode, 

namespace ns = ref::assign_copy;
boost::for_each( ns::csv_anon( x, y, z ), printer(os) );
//outputs (1,0)(NULL,0)(1,5)

In assignment mode, 

T x1, y1, z1;
ns::csv_anon( x1, y1, z1 ).assign( T( b, a ) );
BOOST_ASSERT( y1 == T( b, a ) );

Namespace assign_rebind
Synopsis

Expression

Post condition 

cont[i] = t


&cont[i] == &t

Example

In access mode, 

namespace ns = ref::assign_rebind;
typedef ns::nth_result_of::csv_anon<3, T const>::type result_;
result_ result = ns::csv_anon( x, y, z );
BOOST_ASSERT( &result[ 1 ] == &y );

In assignment mode, 

T w = T( b, a ); result.assign( w );
BOOST_ASSERT( &result[ 1 ] == &w );


[3] Under C++03. Under C++0x, const-ness is, for now, determined by t[0]
