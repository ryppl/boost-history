operator|()

_csv_put

Pointer container

Header 

#include <boost/assign/v2/put/pipe.hpp>

Description

The semantics of operator|(), in this library, emulate those of range adaptors to the extent that they are composable and therefore convenient in conjunction with range algorithms. In the context of range adaptors, expression 

lhs | rhs

maps a range, identified by lhs, according to specifications encapsulated by rhs. In our context, unless otherwise stated, lhs is a Container and the above expression modifies its state according to specifications embodied by rhs, and then returns lhs by reference.

_csv_put

Header 

#include <boost/assign/v2/put/pipe/csv.hpp>

Synopsis

Expression

Result

Side effect

cont | _csv_put( t[0], ....,t[n-1] )

A reference to cont

Puts t[0], ....,t[n-1] in cont

Modifier

The particular side effect is determined by the cont 's category ( Sequence, Associative etc.), and can be found under Keywords in this section.

Miscellanea

In the right hand side of operator|, the result of _csv_put() internally keeps references to its arguments so as to avoid any storage overhead.

Example 

std::vector<int> cont;
boost::for_each(
    cont | as2::_csv_put( -1, 0, 1 ),
    os << bl::_1 << ' '
); //outputs -1 0 1

_put

Header 

#include <boost/assign/v2/put/pipe/functor.hpp>

Introduction

Consider this example 

typedef std::map<const char*, int> cont_;
typedef cont_::value_type type;
cont_ cont;
assert(
    (
        cont | as2::_csv_put(
            type("x", -1),
            type("y", 0),
            type("z", 1)
        )
    )["y"] == 0
);

Instead of explicitly calling type() for each set of arguments, what we'd like, the topic of this section, is to automatically forward to the constructor.

Synopsis

In terms of our notation, for n = 1,

Expression

Result

Side effect

cont | _put(arg[0], ....,arg[k-1])

A reference to cont

Forwards to T(arg[0], ....,arg[k-1]) and puts the result in the container.

The above extends to arbitrary n as follows: 

cont | _put(arg[0][0], ....,arg[0][k[0]-1])...(arg[n-1][0], ....,arg[n-1][k[n-1]-1])

Value-type

The appropriate type, T, is deduced at compile time, from the type of destination container, cont.

Modifier

See comment.

Miscellanea

See comment.

Example 

std::map<const char*, int> cont;
assert(
    (
        cont | as2::_put("z", 1)("x", -1)("y", 0)
    )["y"] == 0
);

Pointer container

This section discusses the side effect of operator|(), with a pointer container as left-argument. Consider this example: 

boost::ptr_vector<int> cont;
boost::for_each(
    cont | as2::_csv_put( -1, 0, 1 ),
    os << bl::_1 << ' '
); // output -1 0 1

Copies of x, y and z are allocated on the heap, and their pointers put into cont, using push_back(), in this case.

put_range

Header 

#include <boost/assign/v2/put/pipe/range.hpp>

Synopsis

Let t[0]...t[n] denote the dereferenced values of range r.

Expression

Result

Implementation 

cont | put_range( r );

A reference to cont 

cont | _put(t[0])...(t[n])


boost::type<Cont>() | put_range( r );

Object of type Cont 

Cont cont; cont | put_range( r );

Note

The second expression is a range adpator

Example 

std::vector<int> v( 3 ); v[0] = -1; v[1] = 0; v[2] = 1;
std::queue<int> fifo; 
assert( ( fifo | as2::_put_range( v ) ).front() == -1 );
assert( fifo.back() == 1 );
