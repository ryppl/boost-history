Anonymous containers

Anonymous containers can be converted to an arbitray containers. Such as tool is particularly relevant when the destination container is an adapter container as they cannot be constructed from a pair of iterators.

It's possible to perform conversion in place, as is sometimes necessary for copy-constructing a container: 

typedef std::queue<T> to_;
to_ to = ref::assign_copy::csv_anon( x, y, z ).convert<to_>();
to.pop();
BOOST_ASSERT( to.front() == y ); // because FIFO

While we can use the same member function, convert<>(), to assign a container, implicit conversion suffices: 

typedef std::deque<T> to_;
to_ to; to = ref::assign_copy::csv_anon( x, y, z );
to.pop_back(); BOOST_ASSERT( to.back() == y );

Adaptor _chain

Here's a case where chaining takes places: 

typedef boost::array<T, 2> from_;
typedef std::stack<T> to_;
from_ from; from[0] = x; from[1] = y;
using namespace adaptor;
namespace ns = ref::assign_copy;
BOOST_ASSERT(
    (
        (
            from | _chain( ns::csv_anon( z ) )
        ).convert<to_>()
    ).top() == z // because LIFO
);

Inter-STL-categories

The library also caters to inter STL categories conversion, with the help of the _convert adaptor. Here's an example: 

typedef boost::array<T, 3> from_;
typedef std::stack<T> to_;
from_ from; from[0] = x; from[1] = y; from[2] = z;
using namespace adaptor;
BOOST_ASSERT(
	( from | convert<to_>() ).top() == z // because LIFO
);
