Overview

This library provides a small set of tools for carrying out operations that can be characterized as a) putting a set of values in a container, or b) iterating through a set of variables. In each case, it allows to code in one sweep, what would ordinarily require repetitive statements and integrates nicely with range algorithms.

For a), a typical call is 

boost::copy( 
	cont | as2::_csv_put(x, y, z), 
	std::ostream_iterator<T>(std::cout," ")
);

It passes cont to an output stream, after putting in it, x, y, and z. How, precisely? In the way the library deems most natural, for example, by calling push_back() if cont is of type std::vector<>. Alternatively, a paramater passing mechanism allows the user to make explicit choices. For example, to request that modifier push_front() be called, 

list | ( as2::_put %  as2::_push_front )( a, b )()( z );

Here, each set of arguments is forwarded to the constructor of T, the value-type of list. Notice that the second value is default constructed. The forwarding mechanism itself, however, could also be overriden, if desired, by passing another parameter. This library makes it easy, through the use of macros, to define custom parameters.

The functionality just described caters primarily to STL containers, but also boost::array<> and pointer containers. In addition, this library defines a constructible-in-place container similar to std::deque<> that shares the same interface as the one above.

To serve the purpose labeled as b), this library generalizes the idea of a reference wrapper to an array (more precisely, boost::array<>) e.g. 

as2::ref::csv_array(x, y, z).assign( T( a, b ) );

A tool for forming the union of ranges that caters to the possibility that at least of them is a reference-array is also provided.
