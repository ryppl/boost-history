Description

An anonymous container of values is akin to a std::deque<> that can be constructed in place.

This section is kept brief because much of the workings of the interface can be deduced from that of the _put adaptor, which is explained in detail. Specifically, they share the csv vs argument forwarding (functor) dichotomy and the parameter passing mechanism, via operator%.

Conversion properties are postponed to the section by that name.

Notice that the result type of csv_anon<T>() and anon<T>() are of the same type. However, only using the former can T be deduced from the arguments [5] and it is known that arg[0][0] is of type T, such that we may label it, instead, t[0], the first two calls to operator() are equivalent anon<T>(_nil)( t[0]). Under these conditions, it is equivalent, but preferable to call csv_anon( t[0] ). ] However, only using second function can create an empty container by passing keyword _nil.

Example

Here's an example using the csv form: 

boost::for_each( csv_anon<T>( x, y ,z ), printer(os) );
// outputs (1,0)(NULL,0)(1,5)

Here's an example using the functor form, and a parameter passed with operator% 

boost::for_each( 
    (anon<T>( _nil ) % _push_front)( a )()( a, b ), 
    printer(os) 
);
// outputs (1,5)(NULL,0)(1,0)


[5] If, in the expression for argument forwarding, k[0 = 1
