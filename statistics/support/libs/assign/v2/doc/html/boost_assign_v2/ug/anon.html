Usage

An anonymous container of values is akin to a std::deque<> that can be constructed in place. This section is kept brief because much of the workings of the interface can be deduced from that of the _put adaptor, which is explained in detail. Specifically, they share the csv vs argument forwarding (functor) dichotomy and the parameter passing mechanism, via operator%.

Here's an example using the csv form: 

boost::for_each( csv_anon<T>( x, y ,z ), printer(std::cout) );
// outputs (1,0)(NULL,0)(1,5)

Here's an example using the functor form, and a parameter passed with operator% 

boost::for_each( 
    (anon<T>( _nil ) % _push_front)( a )()( a, b ), 
    printer(std::cout) 
);
// outputs (1,5)(NULL,0)(1,0)

The overloads of each of the csv and functor forms are limited in such as way as to minimize redundant interfaces. Specifically, if the first argument has type T, is it is preferable to call 

csv_anon( t[0] )(a, b)

rather than 

anon<T>( _nil )( t[0] )(a, b)

as the results are exactly the same. Conversely, the only way to create an empty container is by calling anon<T>( _nil ).
