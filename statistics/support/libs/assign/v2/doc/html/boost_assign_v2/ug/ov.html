Overview

In this section we explain how to get started and give some flavor of the features of this library, which cater to the two kinds of applications hinted at in section Introduction, i.e. working with containers and a set of variables, respectively. These are called the put and ref frameworks, respectively.

Getting started

To be able to use this library, it is necessary to put the directory containing the Boost library in the search path of the C++ compiler and it suffices to declare 

#include <boost/assign/v2.hpp>

The functionality's identifiers are defined in namespace boost::assign::v2, and the convention in this document, is that it is brought into scope with a using declaration.

Put

A typical call is 

boost::copy( 
	cont | _csv_put(x, y, z), 
	std::ostream_iterator<T>(std::cout," ")
);

It passes to output stream std::cout the elements of cont, after putting in the latter the values x, y, and z. How, precisely? In the way the library deems most natural, for example, by calling push_back() if cont is of type std::vector<>. Alternatively, a paramater passing mechanism allows the user to make explicit choices. For example, to request that modifier push_front() be called, 

list | ( _put % _push_front )( a, b )()( z );

Here, each set of arguments is forwarded to the constructor of T, the value-type of list. Notice that the second value is default constructed. If needed, the forwarding mechanism itself could also be overriden, by passing another, adequate, parameter. This library makes it easy, through the use of macros, to define custom parameters.

The functionality just described caters primarily to STL containers, but also boost::array<> and pointer containers. In addition, this library defines a constructible-in-place container similar to std::deque<> that shares the same interface as the one above.

Details of the above and additional features are the topic of section Put.

Ref

This library provides a generalization a reference wrapper to an array (like boost::array<>, not the C-style breed) e.g. 

ref::csv_array(x, y, z).assign( T( a, b ) );

A tool for forming the union of ranges that caters to the possibility that at least of them is a reference-array is also provided.

Notice that some identifiers, such as _put, have prefix _. These are const classes in an anonymous namespace, which we refer to as keywords.
