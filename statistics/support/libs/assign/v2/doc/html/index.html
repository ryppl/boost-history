Preface

This library provides a small set of tools for carrying out operations that can be characterized as taking a set of elements, and putting them in a container. It allows to code in one sweep, what would ordinarily require repetitive statements. More specifically, the destination container can be modified in place through adaptors [1], and, for instance, passed to a range algorithm: 

boost::copy( 
	cont | adaptor::_csv_put(x, y, z), 
	std::ostream_iterator<T>(std::cout," ")
);

The variable aspect of the implementation defaults to one that is deduced from the container's properties, so that the interface is uniform across container categories, such as, in the snippet that follows, an associative container, 

map | adaptor::_put("x", x)("y", y)("z", z);

Alternatively, the user can selectively specify his/her choice of implementation using a parameter passing mechanism: 

list | ( adaptor::_put % ( _fun = f ) % _push_front )( a, b )()( z );

The library supports adding custom parameters, the development of which is facilitated by macros.

While typical clients of the above involve STL containers, the library also caters to boost::array<> and pointer containers. In addition, this library defines two anonymous containers[2], one of values and the other of references[3]. The latter is particularly relevant if the elements are expensive to copy, for modifying them using a range algorithm, or simply assigning them a common value: 

ref::assign_copy::csv_anon(x, y, z).assign( T( a, b ) );

Finally, an adaptor for forming the union of ranges, in a way that that exploits the particular properties of our anonymous container of references, if needed, is provided.


[1] A concept borrowed from range adaptors. Strictly speaking, the object in the right hand side of operator| is an adaptor generator, but we take the liberty to call it by its shorthand, adaptor.

[2] By that we mean constructible in place

[3] It can be thought of as the extension of a reference wrapper to a static array.
