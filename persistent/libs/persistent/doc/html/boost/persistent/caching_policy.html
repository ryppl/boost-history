Struct caching_policy

boost::persistent::caching_policy

Synopsis

// In header: <boost/persistent/cache.hpp>


struct caching_policy {
  // construct/copy/destruct
  caching_policy();
  std::size_t maximum_cache_size;
  std::size_t minimum_cache_size;
  std::size_t block_size;
  posix_time::time_duration timer_resolution;
};

Description

Describes the caching policy of the internal object cache. Objects that are currently not in use are not immediatly removed or saved back to disk but are put in an object cache. When the cache overflows, the objects in the cache that weren't used for the longest time are removed.

  • maximum_cache_size: Maximum number of objects in the object cache. When maximum_cache_size is reached, the cache size is reduced to minimum_cache_size. Note that there is no way to determine the size of an user object, so these settings are measured in numbers of objects. Set the cache size accordingly if large database objects are used. Default: 1000000.

  • minimum_cache_size: Number of objects the cache contains after maximum_cache_size has been reached. Default: 900000

  • block_size: The size of a cache block, in number of objects. Default: 10000

  • timer_resolution: The resolution of the cache timer to be used to determine the time since last access of an object. Default: 1/32 s
