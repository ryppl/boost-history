I/O Manipulator Library

The header boost/io/iomanip.hpp covers C++ I/O stream-manipulating functions (possibly including function templates and function-acting [template] classes).

Rationale

There are several I/O manipulators that can be useful to a general audience besides the ones given in the Standard library. This header supplies some of them.

Header Synopsis

#include <iosfwd>   // for std::basic_ostream, std::basic_istream, and std::basic_ios (declarations)

namespace boost
{
namespace io
{

template < typename Ch, class Tr >
    std::basic_ostream<Ch, Tr> &  newl( std::basic_ostream<Ch, Tr> &os );

template < typename Ch, class Tr >
    std::basic_istream<Ch, Tr> &  skipl( std::basic_istream<Ch, Tr> &is );

template < typename Ch, class Tr >
    std::basic_ios<Ch, Tr> &  resetios( std::basic_ios<Ch, Tr> &s );

class multi_newl;
class multi_skipl;

}
}

Non-flushing Line-break Output Manipulator

template < typename Ch, class Tr >
std::basic_ostream<Ch, Tr> &
boost::io::newl( std::basic_ostream<Ch, Tr> &os );

Effects: inserts a new-line character (sequence), appropiate for the stream's character type and locale, into stream os

Returns: os

Rationale: gives a method of inserting a newline that looks object-based, without either the additional flushing behavior of std::endl, or the background formatting adjustments of using '\n' (or "\n") directly with operator <<

Line-skipping Input Manipulator

template < typename Ch, class Tr >
std::basic_istream<Ch, Tr> &
boost:io::skipl( std::basic_istream<Ch, Tr> &is );

Effects: passes over all the characters in stream is until the first encountered new-line character (sequence) is passed, or the stream's end is reached

Returns: is

Rationale: a counterpart to newl

Stream Formatting State-reset Manipulator

template < typename Ch, class Tr >
std::basic_ios<Ch, Tr> &
boost:io::resetios( std::basic_ios<Ch, Tr> &s );

Effects: resets the formatting attributes of stream s, except locale, to their default values

Returns: s

Postconditions: s.flags() == ( std::ios_base::skipws | std::ios_base::dec ) && s.width() == 0 && s.precision() == 6 && Tr::eq( s.fill(), s.widen(' ') )

Rationale: a quick method to reset all the formatting attributes of a stream, especially after many attribute changes

Multiple-line-break Output Manipulator

Objects from the boost::io::multi_newl class represent manipulators that send multiple consecutive line-breaks to a given output stream. Said manipulators can be temporarily created (in-line, without a name) or created conventionally in advance for multiple insertions or for use as a function object.

#include <ios>      // for std::streamsize
#include <ostream>  // for std::basic_ostream

class boost::io::multi_newl
{
public:
    // Lifetime management
    explicit  multi_newl( std::streamsize count, bool final_flush = false );

    // Operators
    template < typename Ch, class Tr >
    void  operator ()( std::basic_ostream<Ch, Tr> &os ) const;

};

The first (or only) argument of the constructor indicates the number, or count, of line breaks that the manipulator object will insert in a single call. The optional second constructor argument, final_flush, determines whether or not a request to flush the output stream will be done after an insertion (defaults to no flushing).

The function-call operator takes an output stream as its only argument. While the stream os is in a good condition, an unformatted insertion of the new-line character (sequence) occurs count (from the constructor) times. The new-line character (sequence) shall be appropiate for the stream's character type and locale. If mandated by the constructor's final_flush state, then the stream is flushed (if it's still good).

template < typename Ch, class Tr >
std::basic_ostream<Ch, Tr> &
boost::io::operator <<( std::basic_ostream<Ch, Tr> &os, multi_newl const &n );

Effects: calls n( os )

Returns: os

Rationale: alternative to using newl or std::endl multiple times consecutively

Multiple-line-skipping Input Manipulator

Objects from the boost::io::multi_skipl class represent manipulators that pass over many characters until a given number of line-breaks are found in a given input stream. Said manipulators can be temporarily created (in-line, without a name) or created conventionally in advance for multiple extractions or for use as a function object.

#include <ios>      // for std::streamsize
#include <istream>  // for std::basic_istream

class boost::io::multi_skipl
{
public:
    // Lifetime management
    explicit  multi_skipl( std::streamsize count, bool final_sync = false );

    // Operators
    template < typename Ch, class Tr >
    void  operator ()( std::basic_istream<Ch, Tr> &is ) const;

};

The first (or only) argument of the constructor indicates the number, or count, of line breaks that the manipulator object will watch in a single call. The optional second constructor argument, final_sync, determines whether or not a request to synchronize the input stream will be done after an extraction (defaults to no synchronization).

The function-call operator takes an input stream as its only argument. While the stream is is in a good condition, characters will be extracted from the stream and ignored until an expanded new-line character (sequence) is encountered count (from the constructor) times. The new-line character (sequence) expansion shall be appropiate for the stream's character type, traits type, and locale. If mandated by the constructor's final_sync state, then the stream is synchronized (if it's still good).

template < typename Ch, class Tr >
std::basic_istream<Ch, Tr> &
boost::io::operator >>( std::basic_istream<Ch, Tr> &is, multi_skipl const &s );

Effects: calls s( is )

Returns: is

Rationale: alternative to using skipl multiple times consecutively
