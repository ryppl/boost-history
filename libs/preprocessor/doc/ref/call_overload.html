The BOOST_PP_CALL_OVERLOAD variadic macro invokes a non-variadic macro having a given number of parameters.

Usage

BOOST_PP_CALL_OVERLOAD(prefix,...) (v)

Arguments

prefix
The prefix of the non-variadic macro name.
...
Variadic data. The number of variadic data elements, as determined by BOOST_PP_VARIADIC_SIZE, is appended to the prefix to form the output non-variadic macro name, and then the non-variadic macro name is invoked passing this variadic data as paramters.

Remarks

This macro creates a macro name which depends on the number of elements of variadic data using BOOST_PP_OVERLOAD and then invokes that macro passing the variadic data. In this way one can invoke a variadic macro with a variable number of parameters which calls one of a series of non-variadic macros doing very similar things. If such a macro name does not exist taking the specified number of arguments as determined by the number of elements of the variadic data, a preprocessor error will occur.

The macro overcomes a problem using BOOST_PP_OVERLOAD with Visual C++. However, when used with Visual C++ the macro can not be recursively invoked, so BOOST_PP_OVERLOAD must be used instead in a slightly different way. To see how this is done look at the code in boost/preprocesor/tuple/detail/use_overload.hpp. If there is no need to recursively invoke BOOST_PP_OVERLOAD the macro can be used with Visual C++ without any problems. The recursive invocation problem using BOOST_PP_CALL_OVERLOAD does not exist for other compilers tested.

Requirements

Header:  <boost/preprocessor/facilities/overload.hpp>

Sample Code

#include <boost/preprocessor/facilities/overload.hpp>
#include <boost/preprocessor/arithmetic/add.hpp>

#define MACRO_1(number) MACRO_2(number,10)
#define MACRO_2(number1,number2) BOOST_PP_ADD(number1,number2)

#define MACRO_ADD_NUMBERS(...) BOOST_PP_CALL_OVERLOAD(MACRO_,__VA_ARGS__)

MACRO_ADD_NUMBERS(5) // output is 15
MACRO_ADD_NUMBERS(3,6) // output is 9
