fibonacci_heap

This priority queue provides a special update operation that allows the ordering of the items to change. (Many computer science texts define the decreaseKey and increaseKey operations for the same purpose, because some implementations can provide one but not the other. This implementation can do both, so we should keep things simple.) After the ordering criteria (a.k.a. the priority) for item x changes, one must call the member function update(x).

Where Defined

boost/fibonacci_heap.hpp

Template Parameters

Parameter Description Default
IndexedType The type of object stored in the fibonacci_heap.
RandomAccessContainer The type of the underlying container used to implement the fibonacci_heap. std::vector<IndexedType>
Compare The comparison function used to determine whether one IndexedType element is smaller than another. If Compare(x,y) is true, then x is smaller than y. std::less<typename RandomAccessContainer::value_type>
IDMap In order to efficiently find x in the fibonacci_heap, a functor must be provided to map IndexedType to a unique ID, which the fibonacci_heap will then use to map to the location of the item in the underlying container. The ID's generated must be between 0 and N, where N is the value passed to the constructor of fibonacci_heap. identity_property_map

Models

  • Buffer
  • UpdatableBuffer

Type Requirements

  • IndexedType is a model of Assignable.
  • RandomAccessContainer is a model of Random Access Container
  • RandomAccessContainer::value_type is the same type as IndexedType.
  • Compare is a model of Binary Predicate.
  • Compare induces a strict weak ordering, as defined in the LessThan Comparable requirements, on its argument type.
  • IndexedType is convertible to Compare's argument type.

void update(const value_type& x)

Updatable Buffer

Revalidates the fibonacci_heap. A fibonacci_heap is invalidated if, for at least one element y in fibonacci_heap Q such that Compare(x, y) returns true, the instruction:

    while (!Q.empty()) Q.pop();

would remove y before x. This could happen if and only if the ordering that Compare imposed on x and y has changed.

Precondition: x is in the fibonacci_heap.

Complexity

  • The time complexity of the push() and update() operations is O(1) amortized.
  • The time complexity of the pop() operation is O(log n) amortized.
