Replace Algorithms

Overview

The String Algorithms Library contains a set of algorithms for replace and erase operations on sequences. Each algorithm finds a range in the input sequence matching the given criteria and injects a replacement sequence into this range. A replacement sequence does not have to be constant. The implementation allows generic formatting which, for example, is used in variants using regular expressions.

Description

The regular variants are declared in <boost/string_algo/replace.hpp> header and the regex variants are declared in <boost/string_algo/regex.hpp> header.

Rationale

Design and Implementation

The replace algorithms are implemented using a layered structure. The lower layer implements generic substitution of a range in an input sequence with another sequence. This layer take an input in form of find and format functor. These two functors define what is to be replaced and what will be the replacement. The upper layer functions are just wrapping calls to lower layer provided with specific functors. Find functors are shared with the find facility.

Implementation of lower layer is designed to work with a generic sequence, but taking an advantage of specific features if possible. std::vector<T> is considered as a model for a generic sequence.  container_traits<C> class template ( defined in <boost/string_algo/container_traits.hpp> ) is used to determine specific sequence features. Current implementation contains specializations for standard library's std::list<T> , std::basic_string<T> and SGI's  std::rope<T> and std::slist<T>. For compilers supporting partial specialization it is possible to add specialization for a new container by specializing the container_traits<C> class template. For compilers without this feature it is not possible, and default parameter set is used.
