Introduction

The String Algorithm Library provides a generic implementation of various string-related algorithms which are missing in STL. It is an extension to the algorithms library of STL. Algorithms in this library include trimming, case conversion, predicates and find/replace routines. All of them come in different variants so it is easier to choose the best fit for a current need.

Implementation is not restricted to a particular container ( like a std::basic_string ), rather it is as generic as possible. Generalization is not compromising the performance, algorithms are using container specific features when it means a performance gain.

Library also provides a good foundation for generic replace like transformations and searching. Core library algorithms are functor based to be easily customizable. User can provide an implemenation to fit his specific needes. An example of such a customization is regex part of the library ( see <boost/string_algo/regex.hpp> ) and RLE compression example ( see <libs/string_algo/example/rle_example.cpp> )

Build

Whole library is provided in headers. Regex variants of some algorithms, however, are dependant on the Regex++ library. All such an algoritms are separated in <boost/string_algo/regex.hpp> . If this header is used, an application must be linked to the Regex++ library.

Portability

Library has been succesfully compiled and tested with Microsoft Visual C++ 7.0 and GCC 3.2.

Rationale

Copy vs. Mutable

Many algorithms in the library are performing a transformation of the input. Transformation can be done in-place, mutating the input sequence, or a copy of the tranformated input can be created, leaving the input intact. None of these possibilities is supperior to the other, and both have different advantages and dissadvantages. For this reason, both are provided with the library.

Copy versions return a transformed input as a result. Mutable variants return a reference to the input. This allows a simple chaining of transformations within one expression for both ( i.e one can write trim_copy(toupper_copy(s)) as well as trim(toupper(s)) )

Naming

Naming follows the conventions from the Standard C++ Library. If there is a copy and mutable version of the same algorithm,
mutable version is without suffix and copy version has suffix _copy.

Regex

Regulare expressions are essential part of text processing. For this reason, the library provides also regex variants of algorithms. In some places they copy a functionality from the Regex++ library. Goal was to provide complete unified interface for all relevant algorithms in the library.
