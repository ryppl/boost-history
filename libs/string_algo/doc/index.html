Introduction

The String Algorithm Library provide a generic implementation of various string-related algorithms which are missing in STL. It can be seen as an extension to algorithms library of STL. Algorithm in this library includes trimming, case conversion, predicates and find/replace routines. All of them come in different variants so it is easier to choose the best fit for a current need.

Implementation is not restricted to a particular container ( like a std::basic_string ), rather it is as generic as possible. Generalising is not compromising the performance, algorithms are specialized for specific containers, when there is a possibility for optimization.

Many algorithms in this library are implemented using 2 layer structure. There is a generic algorithm in the bottom layer, which is highly paramertic and so not very well suited for general usage. The second layer consist of various specializations for a given algorithm. An example is replace() algorithm. The generic version of replace() accepts a find and format functors as a parameters. The library contains an implementaion of several functors which can be used in this place. raplace_firs() as a second layer algorithm is a simple wrapper around replace that instantiates find_firstF functor and calls a generic version of the replace() to do the job.

Using the String Algorithm Library

General Conventions

 

Trim

Trimming algorithms are included in <boost/string_algo/trim.hpp>

Case Conversion

Case conversion algorithms are included in <boost/string_algo/case_conv.hpp>

Predicates

Predicate algorithms are included in <boost/string_algo/predicate.hpp>

Find

Find algorithms are included in <boost/string_algo/find.hpp>

Replace/Erase

Replace/Erase algorithms are included in <boost/string_algo/replace.hpp> and <boost/string_algo/erase.hpp>
