Find Algorithms

Overview

Find algorithms are used to search for a substring matching a specific criteria. Result is given as a pair of iterators (represented by an instance of iterator_range) pointing into input sequence. Matching criteria is specific per find algorithm. Low-level API provides a generic functor based interface. User-level API encapsulates calls to low-level API using functors provided with the library.

iterator_range class

Description

iterator_range class included in <boost/string_algo/iterator_range.hpp> header ( in boost namespace  ).
Many algorithms working with sequences take a pair of iterators, delimiting a working range, as an arguments. iterator_range class is an encapsulation of a range identified by a pair of iterators. It provides a minimal sequence interface, so it is possible to pass an instance to an algorithm requiring a sequence as an input. This utility class also simplifies a design of an algorithm library, beacuse it is enough to provide just a sequence variant.
