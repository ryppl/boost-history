1
Introduction
1.1. The Problem
The C++ Input/Output library provides a mechanism for customizing the way a class is represented on a linear text stream. There are output operations for the basic data types as well as for std::complex and other data types in the standard library.
Having said that, there are no output operations for any of the STL container types. There is a reason for this: how do we represent the list? Different applications would want different ways for the list to be represented, so there is no easy way to specify a default format. What if you need to output a sub-list? Or have the same format across different container types? Or have different formatting for different objects that have the same container type?
1.2. The Solution
Designing a library that can cope with these problems, as well as others not considered here, while remaining efficient is no easy task. There are other attempts to solve these sorts of problems.
The motivation behind using manipulators is that they are already designed into the C++ I/O library and require little overhead in the output process. They also give the user maximum choice in how the data being outputted is formatted.
1.3. Definitions
1.3.1. ForwardIterator
A type indicating a forward iterator as defined by C++98, that supports the following operations:
a != b compare two iterators for inequality
++i move to the next item
*i access the item data
1.3.2. Container
A container type that has the following:
typename Container::iterator an iterator that is a ForwardIterator type
typename Container::iterator Container::begin() returns the first element in the container
typename Container::iterator Container::end() returns "one-past-the-end" for the container type
1.3.3. Output Facilities
The type is capable of being outputted to a standard ostream:
std::basic_ostream< CharT, Traits > & operator<<
(
std::basic_ostream< CharT, Traits > & os,
const T & type
);
1.3.4. Markers
A sequence of characters used between data items. There are three markers currently used by this library: open sequence; close sequence; and separator.
1.4. Portability
The code has been succesfully tested on:
  1. MSVC 7.0 and MSVC 7.1
  2. Borland C++Compiler 5.5.1 and 5.6.4
  3. GNU G++ 3.x (under the i686-pc-cygwin environment)
  4. GNU G++ 2.95.3-5 (under the i686-pc-cygwin environment) [Note: Type deduction does not work, but the rest of the library does.]
  5. Intel 7.1
Most of the code will now work on MSVC 6.0. Thanks to Jonathan Turkanis for his help porting the code.

Output Manipulators
3.1. boost::io::formatlist
include: boost/outfmt/formatlist.hpp
examples: libs/outfmt/examples/basic-facilities.cpp
3.1.1. Range Output
boost::io::formatlist
(
ForwardIterator first,
ForwardIterator last
);
This manipulator will format the list specified by the range [first, last). It requires that the elements contained in the sequence have output facilities (i.e. define operator<< on a std::basic_ostream< CharT, Traits >).
[Example:
std::vector< int > v;
// ...
std::cout << boost::io::formatlist( v ); // output: [ 3, 6, 9, 12 ]
]
The manipulator conforms to the format object generator guidelines in section [5.1.2.].
3.1.2. Container Output
boost::io::formatlist
(
Container & c
);
This is a variant of the boost::io::formatlist manipulator that takes a sequential container as its argument. It outputs the range [begin(), end()), and requires that the elements in the sequence have output facilities.
[Example:
int i[ 4 ] = { 1, 2, 3, 4 };
std::cout << boost::io::formatlist( i, i + 4 ); // output: [ 1, 2, 3, 4 ]
]
The manipulator conforms to the format object generator guidelines in section [5.1.2.].

4
Formatting Options
examples: libs/outfmt/examples/format-options.cpp
libs/outfmt/examples/formatter.cpp
The formatting facilities are included indirectly when you include other files in the library. They are used in the implementation of the output facilities of the library, including boost::io::formatlist.
These facilities control what is output at the start of a sequence, as a separator between elements and at the end of the sequence.
4.1. Format Traits
The format traits contain the default values for pair types ( boost::io::detail::pair_traits ) and sequence types ( boost::io::detail::format_traits ). These have the following form:
template< typename CharT >
struct format_traits
{
static const CharT open_default = implemenation defined;
static const CharT close_default = implemenation defined;
static const CharT separator_default = implemenation defined;
};
boost::io::detail::pair_traits provides the default formatting:
( ..., ... )
boost::io::detail::format_traits defaults the default formatting:
[ ..., ... ]
Both are defined for char, wchar_t, char * and wchar_t *. If partial template specialisation is supported, it is also specialized for std::basic_string, otherwise std::string and std::wstring are supported.
4.2. Open/Close Formatter
4.2.1. boost::io::openclose_formatter_t
template
<
typename FormatType,
class ReferenceType,
class FormatTraits = boost::io::detail::format_traits< FormatType >
>
class openclose_formatter_t;
boost::io::openclose_formatter_t defines the formatting facilities for wrapping around a output type. It defines the following types:
format_type the character/string type used in the formatting
traits_type a format traits class defined in section [4.1.]
The ReferenceType parameter is used to specify the return type on the format( ... ) function, allowing further access to the derived object. This allows you to apply formatting onto a manipulator, like the example in section [4.2.1.3.].
4.2.1.1. open
typename openclose_formatter_t::format_type openclose_formatter_t::open() const;
Returns the current value for the formatting at the start of the output type.
4.2.1.2. close
typename openclose_formatter_t::format_type openclose_formatter_t::close() const;
Returns the current value for the formatting at the end of the output type.
4.2.1.3. format
ReferenceType & openclose_formatter_t::format
(
format_type o,
format_type c
);
Sets the values for the opening and closing formatting.
[Example:
std::vector< int > v;
// ...
std::cout << boost::io::formatlist( v ).format( "[ ", " )" ); // output: [ 3, 12 )
]
template< typename RT2, class FormatTraits2 >
ReferenceType & openclose_formatter_t::format
(
const openclose_formatter_t< FormatType, RT2, FormatTraits2 > & fmt
);
Copies the values of the opening and closing formatting from one formatter to another, requiring only that they have the same FormatType.
See [4.2.2.] for an example of this.
4.2.2. boost::io::openclose_formatter
template
<
typename FormatType,
class FormatTraits = boost::io::detail::format_traits< FormatType >
>
class openclose_formatter;
boost::io::openclose_formatter is the usable end of this formatter. The user can create an object of this type, set special formatting to it, and then reuse it across different output operations.
[Example:
boost::io::openclose_formatter< char * > fmt( "| ", " |" );
int i[ 4 ] = { 1, 2, 3, 4 };
std::cout << boost::io::formatlist( i, i + 2 ).format( fmt ); // output: | 1, 2 |
]

5
Format Objects: Outputters
5.1. Requirements
5.1.1. Format Objects
5.1.1.1. Funtion Object
An outputter is required to have function object semantics by defining:
template< typename T, class OutputStream >
inline OutputStream & operator()( OutputStream & os, const T & value ) const
(
// ...
return( os );
);
T is the type being outputted using this outputter.
[Note: The type T here is a standin for the actual type being used. For example, the outputter can be implemented to accept std::complex< T >, allowing complex types to be formatted.]
OutputStream is the stream type. This is not std::basic_ostream< CharT, Traits > here to simplify the implementation of the operator (it is easier to type!). If you want to use std::basic_ostream< CharT, Traits > instead, you can.
The operator is const because the outputter is not modified. For state-based formatting that requires modification of the data, like position information, there is a special outputter. See libs/outfmt/examples/john-torjo.cpp for an example of this.
5.1.1.2. Formatting
In order to be chained (when forming complex outputters for nested types), the outputter must define the type format_type to be the string or character type to be used for markers.
In order to allow the user to customise the markers, an outputter should inherit from a formatter. See section [4.] for more information on formatters. [Note: Formatters define the type format_type.]
5.1.2. Format Object Generators
5.1.2.1. Requirements
5.1.2.1.1. name
This is the default format object generator. It uses the char * format type, and the default sub-element outputter used by the outputter.
5.1.2.1.2. name-ex
This generator allows the format type to be specified via function template syntax:
nameex< type >( ... )
[Example:
std::string s = "Hello IO World!";
std::cout << boost::io::formatlistex< char >( s );
]
It uses the default sub-element outputter used by the outputter.
5.1.2.1.3. name-out
This generator allows you to specify the sub-element outputter used by the outputter, allowing you to nest formatting for complex types. It will inherit the format type of the sub-element outputter.
The generator takes an additional parameter: an outputter object.
5.2. Simple Type Outputters
5.2.1. boost::io::basic_output
boost::io::basic_output is used as a generic outputter, providing no specific formatting.
5.2.1.1. boost::io::basicfmt
This generator function creates a boost::io::basic_output object. It does not meets the requirements specified in section [5.1.2.]: it does not have any variants.
5.2.2. boost::io::wrapped_output
boost::io::wrapped_output inherits its formatting from boost::io::openclose_formatter_t, allowing specific formatting to be added around a type. This would be useful for things like outputting a complex type as XML data.
5.2.2.1. boost::io::wrappedfmt
This generator function creates a boost::io::wrapped_output object. It meets the requirements specified in section [5.1.2.].
5.3. N-ary Type Outputters
5.3.1. boost::io::pair_output
examples: libs/outfmt/examples/pair_output.cpp
libs/outfmt/examples/boost_output.cpp
boost::io::pair_output inherits its formatting from boost::io::formatter_t, providing output of std::pair, std::complex, boost::numeric::interval, boost::rational and boost::compressed_pair types.
5.3.1.1. boost::io::pairfmt
This generator function creates a boost::io::pair_output object. It meets the requirements specified in section [5.1.2.].
boost::io::pairfmtout
(
const Outputter & out
);
In this case, both types are outputted using the same outputter.
boost::io::pairfmtout
(
const Outputter1 & out1,
const Outputter2 & out2
);
In this case, each type is outputted using its own outputter, allowing specialist formatting for each type.
5.3.2. boost::io::static_nary_output
examples: libs/outfmt/examples/boost_output.cpp
boost::io::static_nary_output inherits its formatting from boost::io::formatter_t, providing output of boost::math::quaternion and boost::math::octonion types.
5.3.2.1. boost::io::naryfmt
This generator function creates a boost::io::static_nary_output object. It meets the requirements specified in section [5.1.2.].
5.4. Sequence Outputters
5.4.1. boost::io::detail::list_output
boost::io::detail::list_output is a helper class, inheriting its formatting from boost::io::formatter_t. It is used to output sequential lists of the form [first, last).
boost::io::detail::list_output is not technically a format object, since it has the function operator:
template< typename ForwardIterator, class OutputStream >
inline OutputStream & operator()
(
OutputStream & os,
ForwardIterator first,
ForwardIterator last
);
The class is used to help the implementations of boost::io::array_output, boost::io::container_output and boost::io::formatlist_t.
5.4.2. boost::io::array_output
examples: libs/outfmt/examples/output-2D.cpp
libs/outfmt/examples/output-3D.cpp
boost::io::array_output inherits its functionality from boost::io::detail::list_output, supporting the output of C-style arrays of the form const T *.
To facilitate the output of arrays, it defines two special function operators:
operator()( long len );
Specify the range [0, len) to be outputted.
operator()( long off, long len );
Specify the range [off, off + len) to be outputted.
5.4.2.1. boost::io::arrayfmt
This generator function creates a boost::io::array_output object. It meets the requirements specified in section [5.1.2.]. Each form has three sub-variants:
boost::io::arrayfmt( long off, long len )
Construct an array formatter for the range [off, off + len).
boost::io::arrayfmt( long len )
Construct an array formatter for the range [0, len).
boost::io::arrayfmt()
Constructs an array formatter, no range specified.
5.4.3. boost::io::container_output
examples: libs/outfmt/examples/output-2D.cpp
boost::io::container_output inherits its functionality from boost::io::detail::list_output, supporting the output of containers as defined in section [1.3.2.].
5.4.3.1. boost::io::containerfmt
This generator function creates a boost::io::container_output object. It meets the requirements specified in section [5.1.2.].
5.5. Output Manipulators
5.5.1. boost::io::formatlist_t
boost::io::formatlist_t is the class that is used to implement the boost::io::formatlist manipulators. It is a hybrid between a format object and a manipulator. See section [3.1.] for more details.
5.5.2. boost::io::formatob_t
boost::io::formatob_t is the class that is used to implement the boost::io::formatob manipulators. It is a hybrid between a format object and a manipulator, that inherits it's output capabilities from another outputter. See section [3.2.] for more details.
5.6. State Outputters
examples: libs/outfmt/examples/john-torjo.cpp
5.6.1. boost::io::state_output
boost::io::state_output does not inherit from a formatter class, and as such is not a standard format object.
5.6.1.1. State Objects
A state object is a type of function object that has the form:
template< class OutputStream >
inline OutputStream & operator()
(
OutputStream & os
);
A state object should only output state information to the stream.
5.6.1.2. boost::io::statefmt
This generator function creates a boost::io::state_output object. It meets the requirements specified in section [5.1.2.], but has slightly different semantics.
statefmt< StateObject, bool pre >()
This version takes a state object and specifies whether the state object is outputted before ( pre = true ) or after ( pre = false ) the outputter hosted by boost::io::state_output.
statefmtex< FormatType, StateObject, bool pre >()
This version allows the user to specify the format type used by the outputters.
statefmt< StateObject, bool pre >( const Outputter & o )
This version controls the outputter used to format the data type.
